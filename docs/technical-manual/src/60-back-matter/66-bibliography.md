# Bibliography

This bibliography enumerates every external reference cited by the technical manual through tech-manual-v1.0 (Parts I + II + III + IV + V). The companion table in §VI.5 ("BIP Cross-Reference") gives section-by-section cite locations for each BIP; the rows below add author + title + URL + scope-of-use prose.

## Bitcoin Improvement Proposals

For up-to-date BIP texts, see [github.com/bitcoin/bips](https://github.com/bitcoin/bips/blob/master/README.mediawiki). Tagged BIP versions referenced by the manual are stable; the linked README enumerates current status.

- **BIP-38.** Mike Caldwell, Aaron Voisine. *Passphrase-protected private key.* [bip-0038.mediawiki](https://github.com/bitcoin/bips/blob/master/bip-0038.mediawiki). Cited in §V.4.

## Non-BIP standards

- **SLIP-0132.** Trezor / Satoshi Labs. *Registered HD version bytes for BIP-32.* [github.com/satoshilabs/slips/blob/master/slip-0132.md](https://github.com/satoshilabs/slips/blob/master/slip-0132.md). The `zpub`/`ypub`/`Zpub`/`Ypub` extended-key prefix registry; consulted by `mnemonic convert` for prefix-tolerant input and the `--xpub-prefix` output selector. Cited in §III.3, §V.4, and §VI.2 Glossary.

## Academic and protocol references

- **codex32 paper.** Leon Olsson Curr (pseudonym "Pearlwort Sneed"), Andrew Poelstra. *codex32: A Better Way To Back Up Keys.* [codex32.org](https://codex32.org/). The original design analysis for the BCH-over-GF(32) scheme that BIP-93 normatively specifies. Cited in §I.3.
- **miniscript paper.** Andrew Poelstra, Pieter Wuille, Sanket Kanjalkar. *Miniscript: streamlined Bitcoin scripting.* [bitcoin.sipa.be/miniscript](https://bitcoin.sipa.be/miniscript/). The type-system foundations of BIP-379 (Miniscript) underlying md1's bytecode AST. Cited indirectly throughout §II.1, §III.1, §III.2, §V.1.
- **FROST RFC 9591.** Chelsea Komlo, Tim Ruffing, et al. *Flexible Round-Optimized Schnorr Threshold Signatures (FROST).* [rfc-editor.org/rfc/rfc9591](https://www.rfc-editor.org/rfc/rfc9591). The threshold-signature protocol relevant to a potential future fifth m-format point. Cited in §I.2 "Future formats".
- **RUSTSEC-2023-0071.** Marvin Attack timing-side-channel on the `rsa` crate. [rustsec.org/advisories/RUSTSEC-2023-0071](https://rustsec.org/advisories/RUSTSEC-2023-0071). The unpatched advisory deferring BIP-85 `rsa` / `rsa-gpg` applications in `mnemonic derive-child` (§V.4) pending crate stability.

## Reference implementations cited normatively

- **`codex32`** crate v0.1.0 (Andrew Poelstra, CC0). GitHub repository [`rust-codex32`](https://github.com/apoelstra/rust-codex32); crates.io: [docs.rs/codex32](https://docs.rs/codex32). The BIP-93 envelope ms-codec delegates to (the only foreign type that reaches ms-codec's public surface is `codex32::Error`, folded by `From` into `ms_codec::Error::Codex32`). Cited normatively by §II.3, §IV.3, §V.3.
- **`rust-miniscript`** master + crates-io releases (Andrew Poelstra et al.). [docs.rs/miniscript](https://docs.rs/miniscript). The reference miniscript implementation md-codec v0.32's `to_miniscript_descriptor` converter targets, and the descriptor parser the toolkit's BIP-388 wallet-policy pipeline drives. Cited normatively by §II.1, §III.1, §III.2, §III.3, §V.1, §V.4.
- **`rust-bitcoin`** v0.32 (Andrew Poelstra et al.). [docs.rs/bitcoin](https://docs.rs/bitcoin). The `bitcoin::bip32::Xpub`, `bitcoin::Network`, `bitcoin::Address` types that the codec API surfaces (no re-exports — consumers add the crate themselves). Cited in §III.3, §V.1, §V.2, §V.4.
- **`bip39`** (Riccardo Casatta et al.). [docs.rs/bip39](https://docs.rs/bip39). The BIP-39 mnemonic-parsing crate the toolkit and md-codec test fixtures rely on; `bip39::Error` is the inner type of `ToolkitError::Bip39`. Cited in §V.1, §V.4.
- **`getrandom`** crate. [docs.rs/getrandom](https://docs.rs/getrandom). The OS-CSPRNG source for mk-codec's `chunk_set_id` and (transitively) md-codec encoder hardening. Cited in §V.2.
- **`serde` / `serde_json`** (David Tolnay et al.). [docs.rs/serde](https://docs.rs/serde), [docs.rs/serde_json](https://docs.rs/serde_json). The JSON-envelope (de)serializer for the `mnemonic bundle` stdout schema (§V.4.5); `serde_json` is the sole runtime dependency a consumer needs to parse `BundleJson` without depending on `mnemonic-toolkit` itself. Cited in §V.4.
- **`thiserror`** (David Tolnay). [docs.rs/thiserror](https://docs.rs/thiserror). The derive crate generating `Display` for md-codec and mk-codec error enums. Cited in §V.1, §V.2.
