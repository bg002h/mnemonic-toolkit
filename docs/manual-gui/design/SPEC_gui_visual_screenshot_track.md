# SPEC — pixel-faithful PNG screenshots of the 61 GUI forms in the GUI manual (the visual track)

**Status:** draft → R0. **Tier:** cross-repo (mnemonic-gui + mnemonic-toolkit/docs/manual-gui). **Secret hygiene in pixels is a FIRST-CLASS bar.**
**FOLLOWUP:** `manual-gui-visual-screenshot-track` (user-prioritized IMPORTANT). **Recon (authoritative-source-verified, 2026-07-01):** `design/agent-reports/visual-track-recon-2026-07-01.md` — READ IT; every load-bearing fact below cites it.

## 1. Goal
Every one of the **61 GUI subcommand forms** gets an actual **PNG screenshot** (fonts / spacing / colour / layout — the visual layer the structural renders deliberately omit), embedded in the manual's **GUI Forms Reference Part** alongside its structural render, and **gated against drift**. Complements — does NOT replace — the structural track (`verify-examples-gui` and the emit/faithfulness gates are untouched).

## 2. Architecture — the SPLIT-GATE design (forced by recon Q3)
Recon DISPROVED regenerate-anywhere byte-identity for rendered PNGs (backend/driver drift is real and upstream-documented; kittest's own comparison is threshold-based for exactly this reason). Therefore the two repos get DIFFERENT gate primitives, each sound for what it checks:

- **Leg 1 (mnemonic-gui) OWNS RENDERING + re-render determinism.** A kittest **snapshot test suite** renders all 61 forms and compares against PNGs committed in the GUI repo (`tests/snapshots/forms/<tab>-<sub>.png`), using kittest's native dify **threshold-diff at the default 0.6** (near-exact — sub-LSB YIQ tolerance, any single non-AA pixel over fails — yet empirically robust to a full backend swap; recon Q1/Q3). Runs in a **dedicated CI job with a pinned software rasterizer** (lavapipe via pinned Mesa — the wgpu-CI pattern; recon Q2). Env-gated (`GUI_SNAPSHOTS=1`, EARLY-RETURN-SKIP) so `cargo test --workspace` on dev machines / the schema-mirror job is unaffected. Regeneration UX = `UPDATE_SNAPSHOTS=1` (locally, any software rasterizer — CI's pinned gate arbitrates; §5); `.new.png`/`.diff.png`/`.old.png` gitignored. Ships as a **new GUI tag**.
- **Leg 2 (manual-gui) OWNS EMBEDDING + provenance.** The manual commits **copies** of the 61 PNGs (`docs/manual-gui/figures/gui/<tab>-<sub>.png`) and embeds them in the 4 gallery chapters (image + the existing structural render per form section). Its gate — **`verify-figures-gui`** — is a **BYTE-diff of the committed copies against the pinned GUI tag's snapshot files** (read from the lint job's pinned clone — §6; compare; census 61; fail-closed). Byte-diff is VALID here because it checks a file COPY, not a re-render — no Mesa, no cargo, no GPU in manual CI. The pinned-tag provenance chain mirrors how the manual already treats the pinned GUI as ground truth.

Rejected alternatives (recon-grounded): (a) manual-side re-render + threshold gate — duplicates the whole render environment in a docs repo for no added assurance; (c) pinned-container byte-diff as the GUI-side gate — rests on UNVERIFIED cross-CPU LLVM-JIT bit-identity (recon Q3); may be revisited as a tightening if the P0 spike happens to prove it.

## 3. P0 SPIKE (load-bearing; NO further implementation before its result)
On a REAL GitHub `ubuntu-latest` runner: install the pinned software rasterizer (Plan A: `mesa-vulkan-drivers` apt → lavapipe; Plan B: wgpu-CI's pinned Mesa tarball + `VK_DRIVER_FILES`; **Plan C: llvmpipe-GL — `WGPU_BACKEND=gl LIBGL_ALWAYS_SOFTWARE=1`, proven locally by the recon — the gate runs on GL if only C works, corpus generated in the same env, threshold arbitration unchanged; M1**), set `WGPU_BACKEND=vulkan` for A/B, render ≥3 representative mnemonic-gui forms via kittest `wgpu`+`snapshot` — **the set MUST include the TALLEST form (I4)** — and prove: (i) the render succeeds headlessly (lavapipe-Vulkan is the ONE recon-UNVERIFIED residual); (ii) two runs in the same job are byte-identical (recon: same-env byte-stability held locally); (iii) re-runs on **≥3 distinct runner instances** all pass the 0.6 threshold against the first run's PNGs (I2 — one sample is not fleet evidence; at P-phase close the full 61-form suite repeats this). ALSO MEASURE (M2, the plan depends on these): `fit_contents()` vs fixed-size on the tallest form (the I4 sizing fact), full-61 render + job wall-time (cold vs rust-cached), actual corpus size vs the ~3.4 MB estimate. The spike output (adapter string, backend, PNG hashes, threshold results, measurements) persists to `design/agent-reports/`. **STOP fires only if A, B, AND C all fail on runners — the track pauses and the FOLLOWUP is updated (no fabricated fallback).**

## 4. What a screenshot IS (determinism + fixture contract)
- **State:** the SAME canonical on-load state as the structural track — `render_fixture(tab, sub)` (blank `FormState::default()`), rendered through the real form path and settled by the harness (`run_ok`; the GUI's own auto-seed fills defaults). The PNG therefore shows exactly the screen the structural render describes and the faithfulness gate proves (single-fixture coherence across both tracks).
- **Determinism knobs (recon Q5):** kittest already disables cursor blink at construction; animations settle via run-to-quiescence; fonts are crate-embedded (no system fonts); theme = egui's default **dark** (the GUI sets no custom visuals) — **USER-DECIDED 2026-07-01 (M6): dark/faithful**, depicting exactly the on-launch GUI; the light-print tradeoff was surfaced and accepted; captions/Part-intro note the dark theme; a light variant stays out of scope. Fixed `pixels_per_point = 2.0` (crisp print) + a sizing policy — `fit_contents()` vs per-form measured height (the SAME pair as the I4 bar below; m3); the PLAN decides after P0 measures both, and whichever it picks is BOUND by the no-clipping bar.
- **SECRET HYGIENE (first-class — pointed at the ACTUAL pixel channel, I3):** the values that reach pixels come from the schema `default_value` prefill (the auto-seed), NOT the blank fixture — so the assurance must cover that channel. THREE layers: (1) a machine assertion, schema-wide, `flag.secret == true ⇒ flag.default_value.is_none()` (a one-line test over the static schema tables; TRUE at master today — 63/63 secret flags default-None — the assertion keeps it true by machine, not audit); (2) secret widgets render `password(true)`-masked on load regardless (`secret_widget.rs` + slot/tree/widget call-sites) — the independent second layer; (3) the injection INVARIANT: no snapshot test may write any value into a `flag_is_secret` field before rendering. Pixels can't be grepped — the assurance is channel-side + widget-side, stated honestly.
- **NO-CLIPPING ACCEPTANCE BAR (I4):** the real app scrolls its forms (`ScrollArea::vertical()` in main.rs); the harness whole-form render path does NOT scroll; at any fixed viewport, tall forms (bundle, import-wallet, …) would bottom-clip — the PNG would depict a strict SUBSET of what the structural render lists, the census would still pass, and nothing would fail. BAR: **every flag row of the settled canonical form must be visible in the PNG (no viewport clipping)** — the plan's sizing freedom (`fit_contents()` vs per-form measured height) is bounded by this bar. The P0 spike must include the TALLEST form and confirm `fit_contents()` behaves with the non-scrolling whole-form path (expected fine precisely because there is no ScrollArea — measure, don't assume).

## 5. Leg-1 details (mnemonic-gui)
- Dev-dep change only: `egui_kittest = { version = "0.31", features = ["wgpu", "snapshot"] }` — adds dify/colored/getopts/rayon to the DEV graph only; shipped binary graph untouched; the existing AccessKit-only kittest tests (the PR-#24 harness subset, ~58 of the 69 test files) are unaffected (LazyRenderer defers all wgpu work until `render()`; recon Q4; M7). No MSRV bump expected (kittest 1.81; verify dify compiles at 1.88 in P1).
- New `tests/gui_form_snapshots.rs`: enumerate the 61 forms (reuse the PR-#24 enumerator + `render_fixture` + the whole-form render path), `GUI_SNAPSHOTS=1` early-return-skip, one `snapshot_options(...)` per form → `tests/snapshots/forms/<tab>-<sub>.png`, default 0.6 threshold, census 61. `.gitignore` the `.new/.diff/.old` artifacts.
- **New CI job in `build.yml` — fires on EVERY PR + every master push + every `mnemonic-gui-v*` tag push, as a REQUIRED check, NO path filter (I1 ruling).** The P0-proven rasterizer recipe + `GUI_SNAPSHOTS=1 cargo test --test gui_form_snapshots`. Rationale: the manual's byte-copy gate inherits its meaning from THIS job — if it were scheduled/path-filtered, a form-changing PR could merge with stale PNGs, a tag would freeze the staleness, and the manual's byte-gate would go green on screenshots that don't depict the pinned GUI (the schema_mirror-v0.27.x lagging-indicator class, with no downstream catch). `build.yml` already triggers unfiltered on PRs + tags; `Swatinem/rust-cache` bounds the cost. A schedule adds nothing once per-PR firing exists — REJECTED.
- **Ran-at-all tripwire (I1 companion):** the env-gate makes silent skip the dual hole — a mistyped `GUI_SNAPSHOTS` would pass vacuously forever. The CI job MUST prove 61 comparisons executed: kittest writes `{name}.new.png` on every gated-ON comparison, so a post-test step asserts `find tests/snapshots/forms -name '*.new.png' | wc -l` == 61 (or an in-test executed-count assertion). The skip path `eprintln!`s a loud marker (M8).
- **Threshold posture + remediation ladder (I2 ruling — policy, not improvisation):** the gate threshold is kittest's default **0.6** (arithmetic: ±1-LSB all-channel drift ≈ 0.505 < 0.6; the first failure is a ±2-LSB single-channel non-AA pixel; recon: a FULL backend swap produced 0 diffs @0.6). Churn within one pinned rasterizer across heterogeneous runner CPUs is unlikely but not provably zero. If a snapshot fails without an intentional UI change: (1) inspect the `.diff.png` artifact FIRST; (2) if the diff is genuinely invisible, raise the threshold for THAT FORM ONLY via per-test `SnapshotOptions`, with the change + a human visual sign-off recorded in the PR; (3) NEVER blanket-raise the default; (4) if drift is systemic across forms, move regeneration into the pinned environment (wgpu-CI Mesa tarball; `workflow_dispatch` + `UPDATE_SNAPSHOTS` artifact so the corpus equals fleet rendering). Explicitly: a pixel-COUNT budget is NOT available in kittest (egui #5683) and must not be promised.
- **Regeneration UX (M4, honest posture):** contributors regenerate LOCALLY on any software rasterizer (`UPDATE_SNAPSHOTS=1`; the recon proved a full backend swap stays under 0.6, so locally-regenerated bytes pass the pinned CI gate); CI's pinned threshold gate arbitrates. The pinned-env `workflow_dispatch` path from the ladder is the escalation, not the default.
- **Release:** new GUI tag (MINOR: new committed snapshot corpus + test + CI job; app behavior unchanged). ~1–3.4 MB of PNGs committed plain (recon Q6: git-lfs NOT warranted; egui's LFS choice reflects a far larger corpus — and LFS would break the manual's plain-clone copy gate).

HTML uses the Makefile's existing `--self-contained` (align with what exists). PDF size stays sane — expect +~3.4 MB.

## 7. What it catches / non-goals
- **Catches:** any visual change to any form (layout shifts, style/theme changes, widget-appearance regressions, spacing) — at the GUI repo, on the pinned rasterizer, at near-exact threshold; and any manual-figure staleness vs the pinned tag (byte-exact).
- **Non-goals:** the HUMAN UX-review question ("is this layout confusing") — neither structural nor pixel renders answer it; remains a separable FOLLOWUP item. Light-theme variants, interaction/animation shots, non-form chrome (output panel / modal — the documented residual), OS-native window decorations (kittest renders egui content only — no titlebar; state this in the Part intro so readers don't expect a desktop-window look).
