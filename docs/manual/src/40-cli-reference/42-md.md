# `md` (md-cli) reference

The standalone CLI for the md1 format (descriptor-mnemonic /
md-codec). Eight subcommands. Most users will use `mnemonic bundle`
and `mnemonic verify-bundle` instead; `md` is for direct
descriptor-card inspection or when integrating md1 into a non-toolkit
pipeline.

Every subcommand below accepts `--help` (`-h`) for inline help.

---

## `md encode`

Encode a BIP-388 wallet-policy template into one or more md1
backup strings.

### Synopsis

```sh
md encode [OPTIONS] [TEMPLATE]
```

### Flags

| Flag | Purpose |
|---|---|
| `--from-policy <EXPR>` | compile a sub-Miniscript-Policy expression into a template |
| `--context <CTX>` | script context (`tap` / `segwitv0`) for `--from-policy` |
| `--unspendable-key <KEY>` | tap-context fallback hint passed to miniscript's `compile_tr`; defaults to BIP-341 NUMS H-point when omitted (auto-NUMS). Rejected for `--context segwitv0` and when used without `--from-policy` |
| `--path <PATH>` | override the inferred shared derivation path |
| `--key <@i=XPUB>` | concrete xpub for placeholder `@i`; repeatable |
| `--fingerprint <@i=HEX>` | master-key fingerprint for placeholder `@i`; repeatable |
| `--network <NETWORK>` | xpub validation + JSON output labelling |
| `--force-chunked` | force chunked encoding even for short policies |
| `--force-long-code` | force the long BCH code even when the regular suffices |
| `--policy-id-fingerprint` | print the freshly-computed PolicyId fingerprint after the phrase |
| `--json` | emit JSON output |

### Worked example

```sh
md encode 'wpkh(@0/<0;1>/*)'
# md1qqpqqxqxkceprx7rap4t
```

## `md address`

Derive bitcoin addresses from a wallet-policy-mode descriptor (md1
phrases or template + keys).

### Synopsis

```sh
md address [OPTIONS] <PHRASES|--template <TEMPLATE>>
```

### Flags

| Flag | Purpose |
|---|---|
| `--template <TEMPLATE>` | BIP-388 template; requires `--key`; mutually exclusive with positional phrases |
| `--key <@i=XPUB>` | concrete xpub for placeholder `@i`; requires `--template`; repeatable |
| `--fingerprint <@i=HEX>` | master-key fingerprint for placeholder `@i`; requires `--template` |
| `--network <NETWORK>` | xpub validation and address rendering |
| `--chain <CHAIN>` | multipath alternative selector (0 = receive, 1 = change) |
| `--change` | sugar for `--chain 1` |
| `--index <INDEX>` | starting index along the wildcard |
| `--count <COUNT>` | number of consecutive addresses |
| `--json` | emit JSON output |

### Worked example

```sh
md address md1zsxdspq...  # decode + derive address from existing card
```

## `md compile`

Compile a sub-Miniscript-Policy expression into a BIP-388 template.

### Synopsis

```sh
md compile [OPTIONS] <EXPR>
```

### Flags

| Flag | Purpose |
|---|---|
| `--context <CTX>` | script context (`tap` / `segwitv0`) |
| `--unspendable-key <KEY>` | tap-context fallback hint passed to miniscript's `compile_tr`; defaults to BIP-341 NUMS H-point when omitted (auto-NUMS). Rejected for `--context segwitv0` |
| `--json` | emit JSON output |

### Worked examples (v0.17)

```sh
# Single-key tap (key-path-only):
md compile 'pk(@0)' --context tap
# tr(@0)

# Inheritance / timelock pattern (extract wins; @0 is internal key,
# the timelocked branch becomes the script-path leaf):
md compile 'or(pk(@0),and(pk(@1),older(144)))' --context tap
# tr(@0,and_v(v:pk(@1),older(144)))

# 2-of-3 hardware-wallet multisig (auto-NUMS internal key —
# script-path-only spending via multi_a):
md compile 'thresh(2,pk(@0),pk(@1),pk(@2))' --context tap
# tr(50929b74...ce803ac0,multi_a(2,@0,@1,@2))

# Force script-path-only with explicit NUMS (rare; useful for
# and-conjunctions where extraction always fails):
md compile 'and(pk(@0),pk(@1))' --context tap \
    --unspendable-key 50929b74c1a04954b78b4b6035e97a5e078a5a0f28ec96d547bfee9ace803ac0
# tr(50929b74...ce803ac0,and_v(v:pk(@0),pk(@1)))

# Segwitv0 wsh (--unspendable-key not allowed here):
md compile 'multi(2,@0,@1,@2)' --context segwitv0
# wsh(multi(2,@0,@1,@2))
```

## `md verify`

Verify md1 strings re-encode to a given template.

### Synopsis

```sh
md verify --template <TEMPLATE> [OPTIONS] <STRINGS>...
```

### Flags

| Flag | Purpose |
|---|---|
| `--template <TEMPLATE>` | BIP-388 template to verify against |
| `--key <@i=XPUB>` | concrete xpub for placeholder `@i`; repeatable |
| `--fingerprint <@i=HEX>` | master-key fingerprint for placeholder `@i` |
| `--network <NETWORK>` | xpub validation network |
