# `ms` (ms-cli) reference

The standalone CLI for the ms1 format (mnemonic-secret / ms-codec).
Nine subcommands (including the v0.7.0 BIP-93 codex32 K-of-N
[`split`](#ms-split) / [`combine`](#ms-combine) pair). Most users will
use `mnemonic bundle` instead; `ms` is for direct secret-card
inspection or for ms1-only pipelines.

ms1 adopts BIP-93 codex32 directly via the upstream `rust-codex32`
crate (unlike md1 / mk1, which fork the BCH plumbing); `ms` is a
thin wrapper exposing the codex32 single-string mode in a
toolkit-style CLI.

Every subcommand below accepts `--help` (`-h`) for inline help.

> **Recovering a forgotten BIP-39 passphrase.** If you have your entropy
> (an ms1 backup or seed words) but not the BIP-39 passphrase (the
> optional "25th word"), it cannot be brute-forced from the entropy alone.
> A BIP-39 passphrase has no internal verifier — every candidate yields a
> valid-looking wallet — so correctness is only definable against a value
> you already know (an address, xpub, or master-fingerprint), an
> external-derivation oracle that `ms` does not perform (`ms` only
> encodes/decodes entropy). An external open-source tool does exactly
> this: [**btcrecover**](https://github.com/3rdIteration/btcrecover)
> (maintained fork; [original](https://github.com/gurnec/btcrecover))
> searches passphrase candidates and confirms each by deriving an address
> / xpub / master-fingerprint at common default paths and matching your
> known value. Pointer current as of 2026-05-25; run untrusted recovery
> tools offline, on an air-gapped machine. This mirrors the `ms --help`
> footer.

## `ms encode`

Encode a BIP-39 mnemonic (or hex entropy) as an ms1 string.

### Synopsis

```sh
ms encode [OPTIONS] <--phrase <PHRASE> | --hex <HEX>>
```

### Flags

| Flag | Purpose |
|---|---|
| `--phrase <PHRASE>` | BIP-39 mnemonic; `-` reads from stdin |
| `--hex <HEX>` | hex-encoded entropy bytes (16/20/24/28/32 bytes = 32/40/48/56/64 hex chars) |
| `--language <LANGUAGE>` | BIP-39 wordlist for the input phrase; ignored under `--hex` |
| `--no-engraving-card` | suppress the stderr engraving-card layout |
| `--group-size <N>` | mstring display grouping: insert a separator every N characters in the emitted `ms1` string(s); `0` = unbroken (default 5). Display only — `--json` stays unbroken. Also on `ms split` (per share) and `ms combine --to ms1`. Separator-stripping on intake means grouped/unbroken cards both re-ingest. |
| `--separator <space\|hyphen\|comma>` | the grouping separator for `--group-size` (default `space`); keyword or the literal `-` / `,` or a space. |
| `--json` | emit a single JSON object on stdout |

### `entr` vs `mnem` payload kind (auto-routing)

`ms encode` **auto-routes** to one of two ms1 payload kinds based on the input:

- **`entr`** (byte-identical with ms-cli v0.1): produced for English phrases (`--language english`, the default) and for raw entropy (`--hex`). The payload carries only the entropy bytes. This is the classic form and is compatible with all ms-codec versions.
- **`mnem`** (ms-cli v0.2 / ms-codec 0.3.0+): produced automatically for any non-English `--phrase` (i.e. `--language` is anything other than `english`). The payload carries a one-byte language tag alongside the entropy, so that a decoder can recover the phrase in the **original wordlist language** without the caller needing to specify `--language` at decode time.

This solves the **§6.3 non-English seed footgun**: a BIP-39 seed is PBKDF2 over the *language-specific* mnemonic string, so the same entropy decoded with the wrong language yields a different wallet. With `mnem` encoding the language is preserved on the wire; `ms decode` and `ms inspect` read it back automatically.

> **Note:** `entr` ms1 strings from English sources are **byte-identical** before and after the v0.2 upgrade — existing English backups are unaffected.

### Worked example

```sh
# English phrase → `entr` ms1 (classic, byte-identical with v0.1)
ms encode --phrase "abandon abandon abandon abandon abandon abandon abandon abandon abandon abandon abandon about"
# ms10entrsqqqqqqqqqqqqqqqqqqqqqqqqqqqqcj9sxraq34v7f

# Non-English phrase → `mnem` ms1 (language preserved on wire, v0.2+)
ms encode --phrase "あいこくしん あいこくしん … あおぞら" --language japanese
# ms10entrsqgqsqqqqqqqqqqqqqqqqqqqqqqqqqj9tawneveyd9j
```

## `ms decode`

Decode an ms1 string back to its BIP-39 mnemonic and entropy bytes.

### Synopsis

```sh
ms decode [OPTIONS] <STRING>
```

### Flags

| Flag | Purpose |
|---|---|
| `--language <LANGUAGE>` | BIP-39 wordlist for the output phrase |
| `--json` | emit JSON output |

### `mnem` decode behavior

When decoding a **`mnem`** ms1 string (produced by `ms encode` for non-English phrases, ms-cli v0.2+), the wordlist language is read directly from the wire payload. The `--language` flag is **not required** — the decoded phrase is automatically emitted in the language stored in the card. If `--language` is supplied and disagrees with the on-wire language, the decoder uses the on-wire value (the on-wire language is authoritative).

For classic **`entr`** ms1 strings, `--language` selects the output wordlist (default `english`; annotated `DEFAULT` on stdout and stderr when defaulted).

### Worked example

```sh
# entr ms1: `--language` selects output wordlist (default english)
ms decode ms10entrsqqqqqqqqqqqqqqqqqqqqqqqqqqqqcj9sxraq34v7f
# phrase: abandon abandon abandon … about
# entropy (hex): 00000000000000000000000000000000

# mnem ms1: language read from wire — no --language needed
ms decode ms10entrsqgqsqqqqqqqqqqqqqqqqqqqqqqqqqj9tawneveyd9j
# phrase: あいこくしん あいこくしん … あおぞら
# language: japanese (12 words)
# entropy (hex): 00000000000000000000000000000000
```

## `ms derive`

Derive the **master fingerprint** (the cheapest "did I recover the right seed?" check) and, with `--template`, an **account xpub** for watch-only setup. Read-only public derivation — **no master seed, root xprv, or private keys on stdout, and `ms` never signs** (use the toolkit's `mnemonic convert` for secret-bearing outputs).

The BIP-39 seed is `PBKDF2` over the *language-specific* mnemonic string, so the master fingerprint/xpub depend on `--language` — when it is omitted, `english` is used and annotated `DEFAULT` (stdout + stderr), exactly like `ms decode`.

### Synopsis

```sh
ms derive [<MS1>] [--hex <HEX> | --phrase <WORDS>] [--template <bip44|bip49|bip84|bip86>] \
          [--account <N>] [--network <mainnet|testnet>] [--passphrase <V> | --passphrase-stdin] \
          [--language <L>] [--json]
```

The entropy comes from one of: a positional `ms1` string (stdin via `-` or omitted), `--hex` raw entropy, or `--phrase` a BIP-39 mnemonic. Without `--template` only the master fingerprint is emitted.

### Flags

| Flag | Purpose |
|---|---|
| `<MS1>` | ms1 string (positional); `-` or omitted reads from stdin |
| `--hex <HEX>` | raw entropy (16/20/24/28/32 B), alternative to the ms1 |
| `--phrase <WORDS>` | a BIP-39 mnemonic, alternative to the ms1 |
| `--template <T>` | `bip44`/`bip49`/`bip84`/`bip86` — emit an account xpub at `m/<purpose>'/<coin>'/<account>'` |
| `--account <N>` | account index for `--template` (default 0) |
| `--network <NET>` | `mainnet` (default) or `testnet`; sets the coin-type + xpub/tpub serialization |
| `--passphrase <V>` | BIP-39 passphrase |
| `--passphrase-stdin` | read the BIP-39 passphrase from stdin (conflicts with `--passphrase`) |
| `--language <L>` | BIP-39 wordlist (load-bearing; default `english`, annotated `DEFAULT`) |
| `--json` | emit JSON output |
| `--help` | print help |

### Worked example

```sh
ms derive ms10entrsqqqqqqqqqqqqqqqqqqqqqqqqqqqqcj9sxraq34v7f --template bip84
# master_fingerprint:  73c5da0a
# account_path:        m/84'/0'/0'
# account_xpub:        xpub6CatWdiZiodmUeTDp8LT5or8nmbKNcuyvz7WyksVFkKB4RHwCD3XyuvPEbvqAQY3rAPshWcMLoP2fMFMKHPJ4ZeZXYVUhLv1VMrjPC7PW6V
# language:            english (DEFAULT)
```

JSON mode emits `{ "schema_version":"1", "master_fingerprint", "network", "account_path"?, "account_xpub"?, "language", "language_defaulted" }` (the `account_*` fields are present only with `--template`).

## `ms split` {#ms-split}

Split a secret (BIP-39 mnemonic or hex entropy) into N **BIP-93 codex32
K-of-N shares**. Any K of the N shares recombine to the original via
[`ms combine`](#ms-combine), using codex32's native `threshold(k)` +
`index` Shamir mechanism over `GF(32)`. The whole N-share SET is
secret-equivalent — a `PrivateKeyMaterial` stderr advisory is emitted —
so engrave each share on its own backup medium. Bounds:
**2 ≤ K ≤ N ≤ 31**.

A non-English `--phrase` splits as a `mnem` share-set so the BIP-39
wordlist language survives the split; an English phrase or `--hex`
entropy splits as a plain `entr` share-set. The toolkit front-end is
[`mnemonic ms-shares split`](#mnemonic-ms-shares).

### Synopsis

```sh
ms split (--phrase <PHRASE> | --hex <HEX>) --threshold <K> --shares <N> \
         [--language <L>] [--json]
```

### Flags

| Flag | Purpose |
|---|---|
| `--phrase <PHRASE>` | BIP-39 mnemonic to split; `-` reads from stdin |
| `--hex <HEX>` | hex-encoded entropy to split (16/20/24/28/32 B = 32/40/48/56/64 hex chars), alternative to `--phrase` |
| `--threshold <K>` | threshold K — minimum shares needed to recombine (2..=9) |
| `--shares <N>` | total shares N to produce (K ≤ N ≤ 31) |
| `--language <L>` | BIP-39 wordlist for the input phrase; ignored under `--hex`. A non-English wordlist produces a `mnem` share-set (default `english`) |
| `--json` | emit a single JSON object on stdout (`{ shares, k, n, id, kind, language? }`) instead of multi-line text |
| `--help` | print help |

### Worked example

```sh
ms split --hex 00000000000000000000000000000000 -k 2 -n 3
# 3 ms1-format codex32 shares (one per line), threshold digit `2`,
# a shared random 4-char identifier, distinct non-`s` indices.
```

Recombine any 2 with [`ms combine`](#ms-combine).

## `ms combine` {#ms-combine}

Recombine K-of-N codex32 shares (from [`ms split`](#ms-split)) into the
original secret. Supply K (or more) distributed shares with distinct
indices; the recovered secret is emitted per `--to`. The
secret-carrying share at index `s` is NEVER a valid input (it is
rejected). The recovered output is private key material (a
`PrivateKeyMaterial` stderr advisory is emitted). The toolkit front-end
is [`mnemonic ms-shares combine`](#mnemonic-ms-shares).

### Synopsis

```sh
ms combine <SHARES>... [--to <phrase|entropy|ms1>] [--json]
```

### Flags

| Flag | Purpose |
|---|---|
| `<SHARES>...` | the distributed share strings to recombine (positional; K or more, distinct indices) |
| `--to <T>` | output form for the recovered secret (default `phrase`): `phrase` (BIP-39 mnemonic — `mnem` → wire language, `entr` → English), `entropy` (raw entropy hex), or `ms1` (a single unshared ms1 string) |
| `--json` | emit a single JSON object on stdout instead of text |
| `--help` | print help |

### Worked example

```sh
ms combine <share1> <share2>              # recover the BIP-39 phrase
ms combine <share1> <share2> --to ms1     # recover a single ms1 string
ms combine <share1> <share2> --to entropy # recover raw entropy hex
```

Fewer than K shares → a "threshold not passed" refusal; a repeated
index → a "repeated index" refusal; the secret share at index `s` → a
"secret share supplied to combine" refusal.

## `ms inspect`

Inspect an ms1 string's structural fields and decoder verdict
(debug-style verbose output).

### Synopsis

```sh
ms inspect [OPTIONS] <STRING>
```

### Flags

| Flag | Purpose |
|---|---|
| `--json` | emit JSON output |

### `kind` and `language` fields

`ms inspect` reports a `kind` field distinguishing the ms1 payload variants:

- `kind: entr` — classic entropy-only payload (ms-cli v0.1 / ms-codec 0.1–0.2).
- `kind: mnem` — language-tagged payload (ms-cli v0.2+ / ms-codec 0.3.0+); the `language` field is also present, naming the stored wordlist (e.g. `japanese`).
- `kind: share` — a BIP-93 codex32 K-of-N share (ms-cli v0.7.0+; from [`ms split`](#ms-split)). Its threshold digit is `2..9` (a single-string ms1 is the unshared sentinel `0`). For a share, `ms inspect` reports `kind: share` plus the `threshold`, `id`, and `index` fields and exits 0 — it does NOT print a `threshold-not-zero` length/decode FAIL. The `prefix_byte` / payload-kind fields are SUPPRESSED for a share: a distributed share's leading data byte is an interpolated `GF(32)` value, not a meaningful payload prefix, and must not be interpreted as one. Recombine ≥K shares with [`ms combine`](#ms-combine).

`entr` strings do not carry a `language` line in `ms inspect` output.

## `ms repair`

BCH error-correct a single ms1 string (`ms-cli v0.4.0`+). Wraps
`ms_codec::decode_with_correction` (ms-codec v0.2.0+) and renders a
single-chunk repair report. The string is corrected within the
`BCH(93,80,8)` `t=4` capacity (up to four substitution errors).

`ms repair` is the per-codec sibling of toolkit's `mnemonic repair`
(see `41-mnemonic.md` `## mnemonic repair`). The two surfaces share
the same `RepairJson` envelope schema byte-exact (cross-CLI parser
reuse — D27 of the toolkit v0.22.x follow-ups cycle plan).

Single-HRP context: no `--hrp` flag. ms1 is single-chunk by codex32
specification (HRP `ms` is always single-string `BCH(93,80,8)`); no
variadic positional.

### Synopsis

```sh
ms repair [OPTIONS] --ms1 <MS1>
```

### Flags

| Flag | Purpose |
|---|---|
| `--ms1 <MS1>` | ms1 string to attempt to repair; use `-` to read the string from stdin (a single line) |
| `--json` | emit a single JSON envelope on stdout instead of the text-form report; schema byte-matches `mnemonic repair --json`'s `RepairJson` shape |
| `--help` | print help |

### Exit codes

| Code | Meaning |
|---|---|
| `0` | input already valid; no correction applied; echoed unchanged. |
| `4` | **`VERIFY-ME` Candidate** (Cycle F, `ms1-repair-demote-to-candidate` — a funds fix) — at least one substitution corrected; stdout = repair report + corrected string; stderr carries an UNVERIFIED advisory. **Never exit 5** — see below. |
| `2` | unrepairable (BCH-uncorrectable: > 4 substitution errors, or structural ms1 error before correction could run). |
| `1` | I/O error or other generic failure. |

**No self-verification, no exit 5 (Cycle F funds fix).** `ms1` encodes
raw BIP-39 entropy as a single codex32 string — a bearer secret with no
cross-chunk hash, no fingerprint, and no internal redundancy beyond the
BCH checksum itself. A bounded-distance (≤4-error) substitution
correction is provably the original string, but beyond that bound the
correction can still *succeed* while **aliasing to a DIFFERENT, valid
seed** — and unlike `mk1`/`md1`, there is nothing else to catch it.
Before this cycle, `ms repair` reported any correction as a confident
exit-`5` `REPAIR_APPLIED`; it now **always** demotes a touched
correction to an exit-`4` Candidate, with a stderr advisory recommending
the user confirm the derived address/xpub against a known-good copy
before trusting it. `ms repair` has no `--max-indel` flag (unlike
`mnemonic repair --ms1 --max-indel`, which keeps a UNIQUE full-checksum
indel recovery at exit `5` — a genuinely different, stronger-verified
mechanism; see `41-mnemonic.md`'s `mnemonic repair` chapter), so exit
`5` is unreachable from this binary entirely. BIP-93 recommends
confirming a corrected codex32 string before relying on it; this
demotion operationalizes that recommendation. The **principled rule
across all four CLIs** (D26 of the v0.22.x follow-ups cycle, refined by
Cycle E + Cycle F): exit-5 `REPAIR_APPLIED` means a correction is
**verified now** or **verifiable-by-reassembly later** (mk1/md1's
cross-chunk structure); exit-4 `VERIFY-ME` means a substitution
correction spent the checksum's error-detection budget with **no
self-oracle** — `ms1`'s case, always.

### Worked example

```sh
# A valid ms1 chunk with one character corrupted (position 17 'q' → 'z'):
ms repair --ms1 ms10entrsqqqqqqqqqqqzqqqqqqqqqqqqqqqqcj9sxraq34v7f
```

### D9 secret-on-stdout advisory

Because the corrected output carries the ms1 (which encodes BIP-39
entropy), the D9 secret-on-stdout advisory fires: a stderr line
recommending the user pipe the output to a file or to an encryption
tool to avoid scrollback exposure. This advisory is informational and
does not affect the exit code — it fires alongside, not instead of, the
UNVERIFIED Candidate advisory above.

### `--json` verdict field (Cycle F)

`ms repair --json`'s `RepairJson` envelope gained a `verdict` field
(`"blessed"` for a clean input, `"candidate"` for a touched correction),
at the identical field position as `mnemonic repair --json`'s envelope
(the D27 byte-match invariant this chapter's intro references).

## `ms gen-man` (v0.13.0) {#ms-gen-man}

Emit roff man pages for the whole `ms` CLI tree into a directory. The pages are
generated directly from the compiled clap `Command` tree (`clap_mangen`), so
they are binary-faithful by construction. One page per (nested) subcommand is
written, hyphen-joined: `ms.1` (root), `ms-encode.1`, `ms-split.1`, and so on.
`scripts/install.sh` invokes this after `cargo install` to drop pages into the
user manpath (no sudo).

### Synopsis

```sh
ms gen-man --out <DIR>
```

### Flags

| Flag | Meaning |
|---|---|
| `--out <DIR>` (required) | Directory to write the `*.1` man pages into (created if absent). |
| `--help` | Print help and exit. |

### Exit codes

| Condition | Exit |
|---|---|
| success | `0` |
| output-dir create / write I/O error | `1` |
