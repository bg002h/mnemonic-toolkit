**Date:** 2026-05-05
**Predecessor:** v0.3.0 SHIPPED (tag `mnemonic-toolkit-v0.3.0`).

## Context

v0.3.0 deferred `tr(@0, sortedmulti_a(...))` to v0.4-cross-repo because rust-miniscript v13.0.0 (the pinned crate) lacks a parser for the fragment.

Subsequent investigation found upstream PR #910 ("Add support for sortedmulti_a") merged 2026-04-03 and PR #915 ("refactor: remove SortedMultiVec and use Terminal::SortedMulti") merged 2026-04-04. Master HEAD `95fdd1c5773b` (commit date 2026-05-02) has both. **No published crate release contains them yet** — latest crate tag is still `13.0.0`.

A read-only build experiment (2026-05-05) confirmed that `[patch.crates-io] miniscript = { git = "...", rev = "95fdd1c5773b" }` + a 3-change walker refactor produces a buildable, test-clean toolkit with end-to-end `tr(@0, sortedmulti_a(...))` support. v0.2 + v0.3 fixture matrices preserved byte-identically. Smoke test emitted a valid schema-3 bundle with `Tag::SortedMultiA` + threshold=2 multisig metadata.

This plan ships that experiment as v0.3.1.

## Walker refactor (3 mechanical changes — all in `parse_descriptor.rs`)

### Change 1 — `walk_wsh` simplification

**Before** (v13.0.0 API):
```rust
fn walk_wsh(w, km) -> Result<Node, ToolkitError> {
    let inner = walk_wsh_inner(w, km)?;
    Ok(wrap_children(Tag::Wsh, inner))
}

fn walk_wsh_inner(w, km) -> Result<Node, ToolkitError> {
    match w.as_inner() {
        WshInner::Ms(ms) => walk_miniscript_node(ms, km, false),
        WshInner::SortedMulti(sm) => build_multi_node(Tag::SortedMulti, sm.k(), &sm.pks().iter().collect::<Vec<_>>(), km),
    }
}
```

**After** (master API; `WshInner` enum removed by PR #915):
```rust
fn walk_wsh(w, km) -> Result<Node, ToolkitError> {
    // Wsh::as_inner() returns &Miniscript<Pk, Segwitv0>; SortedMulti now
    // surfaces as Terminal::SortedMulti inside the inner Ms.
    let inner = walk_miniscript_node(w.as_inner(), km, /*tap=*/ false)?;
    Ok(wrap_children(Tag::Wsh, inner))
}
```

`walk_wsh_inner` is **deleted** (it had only one production call site).

### Change 2 — `walk_sh` arm cleanup + `ShInner::Wsh` call-site update (architect I-1)

**Before:**
```rust
let inner = match s.as_inner() {
    ShInner::Wsh(w) => Node {
        tag: Tag::Wsh,
        body: Body::Children(vec![walk_wsh_inner(w, km)?]),
    },
    ShInner::Wpkh(wp) => ...,
    ShInner::Ms(ms) => walk_miniscript_node(ms, km, false)?,
    ShInner::SortedMulti(sm) => build_multi_node(Tag::SortedMulti, sm.k(), ...)?,
};
```

**After** (`ShInner::SortedMulti` variant removed; `walk_wsh_inner` no longer exists):
```rust
let inner = match s.as_inner() {
    ShInner::Wsh(w) => Node {
        tag: Tag::Wsh,
        body: Body::Children(vec![walk_miniscript_node(w.as_inner(), km, /*tap=*/ false)?]),
    },
    ShInner::Wpkh(wp) => ...,
    ShInner::Ms(ms) => walk_miniscript_node(ms, km, /*tap=*/ false)?,
};
```

Both the `walk_wsh_inner(w, km)?` call site INSIDE `ShInner::Wsh` AND the `ShInner::SortedMulti` arm are patched. Architect I-1 explicitly named the `ShInner::Wsh` call-site update; this plan honors that.

### Change 3 — `walk_miniscript_node` adds two new `Terminal` arms (architect I-2)

Add immediately after the existing `Terminal::Multi` and `Terminal::MultiA` arms, using the **same iteration shape** as the existing arms:

```rust
Terminal::SortedMulti(thresh) => build_multi_node(
    Tag::SortedMulti,
    thresh.k(),
    &thresh.data().iter().collect::<Vec<_>>(),
    km,
),
Terminal::SortedMultiA(thresh) => build_multi_node(
    Tag::SortedMultiA,
    thresh.k(),
    &thresh.data().iter().collect::<Vec<_>>(),
    km,
),
```

The `&thresh.data().iter().collect::<Vec<_>>()` pattern matches the existing `Terminal::Multi` / `Terminal::MultiA` arms (architect I-2). `thresh.data()` yields `&DescriptorPublicKey`; `iter().collect()` produces `Vec<&DescriptorPublicKey>`; `&` references that slice for `build_multi_node(_, _, &[&DescriptorPublicKey], _)`.

## Tests

### Unit tests (in `parse_descriptor.rs::tests`)

Add 2 round-trip tests:

```rust
#[test]
fn arm_sorted_multi_via_wsh() {
    // wsh(sortedmulti(2,@0,@1)) post-#915 surfaces as Terminal::SortedMulti
    // inside the Wsh's inner Ms (was WshInner::SortedMulti pre-#915).
    let inner = wsh_inner("wsh(sortedmulti(2,@0/<0;1>/*,@1/<0;1>/*))");
    assert_eq!(inner.tag, Tag::SortedMulti);
    let Body::Variable { k, children } = inner.body else { panic!() };
    assert_eq!(k, 2);
    assert_eq!(children.len(), 2);
}

#[test]
fn arm_sorted_multi_a_via_tap() {
    // tr(@0, sortedmulti_a(2,@0,@1)) — the v0.3.1 unblock target.
    let root = parse_and_walk(
        "tr(@0/<0;1>/*,sortedmulti_a(2,@0/<0;1>/*,@1/<0;1>/*))",
        ScriptCtx::MultiSig,
    );
    let Body::Tr { tree, .. } = &root.body else { panic!() };
    let leaf = tree.as_ref().unwrap();
    assert_eq!(leaf.tag, Tag::SortedMultiA);
    let Body::Variable { k, children } = &leaf.body else { panic!() };
    assert_eq!(*k, 2);
    assert_eq!(children.len(), 2);
}
```

### Integration / regression test (architect L-4)

Add to `crates/mnemonic-toolkit/src/parse_descriptor.rs::tests`:

```rust
#[test]
fn descriptor_tr_sortedmulti_a_matches_template_tr_sortedmulti_a_md1() {
    // Wire-bit-identical regression: descriptor-mode tr(@0, sortedmulti_a(...))
    // should produce md1 byte-identical to template-mode --template tr-sortedmulti-a
    // for matching keys/cosigners. Confirms the new walker arm produces the same
    // Tag::SortedMultiA tree the template encoder has been producing since v0.3.0.
    use crate::synthesize::{synthesize_multisig_full, /* ... */};
    use crate::template::CliTemplate;
    let mnemonic = bip39::Mnemonic::parse_in(bip39::Language::English, TREZOR_24).unwrap();
    let template_bundle = synthesize_multisig_full(
        &mnemonic, "", CliNetwork::Mainnet, CliTemplate::TrSortedMultiA,
        2, 2, 0, MultisigPathFamily::Bip48, false,
    ).unwrap();
    let fp_hex = trezor_master_fp_hex();
    let descriptor = format!(
        "tr(@0[{fp_hex}/48'/0'/0'/2']/<0;1>/*, \
            sortedmulti_a(2, @0[{fp_hex}/48'/0'/0'/2']/<0;1>/*, \
                            @1[{fp_hex}/48'/0'/0'/2']/<0;1>/*))"
    );
    let descriptor_bundle = parse_synth_descriptor(&descriptor, CliNetwork::Mainnet);
    assert_eq!(
        descriptor_bundle.md1, template_bundle.md1,
        "md1 must be byte-identical for descriptor-mode sortedmulti_a vs template TrSortedMultiA"
    );
}
```

This is the **strongest correctness signal** per architect L-4 — confirms the new `Terminal::SortedMultiA → Tag::SortedMultiA` arm produces the same wire output as the v0.3.0-shipped template-mode synthesizer.
