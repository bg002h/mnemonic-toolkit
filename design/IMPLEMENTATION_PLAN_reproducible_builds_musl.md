## P4 — remap-EXERCISING CI self-rebuild drift gate

**GOAL.** A continuous CI drift gate that **keeps `--remap-path-prefix` under test release-over-release** by rebuilding at a path **DISTINCT** from the release-build path and asserting byte-equality to the PUBLISHED `.tar.gz` — plus a remap-off NEGATIVE test that REDs the gate (R0-r3-I1). A same-path same-container rebuild would prove only container-determinism and let the remap silently regress (both real paths identical → nothing to remap-divergently) — the exact false-assurance failure mode of the stale md slug.

**FILES created/changed.**

- **`.github/workflows/reproducible-musl-build.yml`** (EXTEND — the `repro-drift` job). After a release publishes, CI rebuilds in the pinned container by running the §5 **two-path build** (`/build-a/src` + `/build-b/src`, both remapped to `/build`) — a path DISTINCT from the release `/build/src` — and asserts BOTH: (a) the two legs equal each other, AND (b) **each leg's `.tar.gz` SHA256 equals the published `SHA256SUMS.<arch>`**. So byte-identical-to-the-released-asset is achieved from a build at a path DISTINCT from the one that produced it (item §3.14). Fail loudly on any drift. **PRECONDITION (R0-r1-I-release-rehome): assertion (b) is meaningful ONLY because P1/P3 re-homed the release `musl-binaries` build inside the SAME `@sha256:<BUILT-DIGEST>` container at `/build/src` with the identical ENV recipe. P4 therefore validates the closed loop, not an aspiration; do NOT enable P4 until the P1/P3 release-job re-home has landed and GREENed.**
- **Negative-test step** (TDD, committed alongside): a deliberately remap-OFF run (drop `CARGO_BUILD_RUSTFLAGS` / empty it) MUST **RED** the gate (the two distinct-path builds diverge + diverge from the published hash). This proves the gate actually exercises the remap rather than trivially matching. The negative run is gated behind a manual/`workflow_dispatch` input or a `continue-on-error` expectation-inverted step so it is a standing proof, not a release blocker.
- **Per-repo wiring** — the drift gate runs in all 4 CLI repos (centralized via the toolkit reusable workflow if F6 landed in P3; else per-repo).

**CI GATE — `repro-drift` job (the assertion that proves P4):**
