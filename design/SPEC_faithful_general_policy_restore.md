# SPEC — PART 1: faithful general-policy `restore --md1` reconstruction

**Cycle:** toolkit **MINOR** (new reconstruction capability) · **Source SHA:** `5d599f7` · **Recon:** `design/RECON_faithful_general_policy_restore.md` + `design/agent-reports/fragment-backup-restore-review-2026-06-11.md`.
**Fixes:** C1 — `restore --md1` SILENTLY reconstructs a different wallet (lock condition dropped) for general `wsh`/`sh(wsh())` policies whose keys sit inside `multi()`/`sortedmulti()` (verified: `wsh(and_v(v:multi(2,…),older(4032)))` → restores `wsh(multi(2,…))`, the `older` gone, exit 0, false "verified" banner, wrong importable payload).

## Root cause (confirmed)
`restore.rs:839` computes `ms0 = md_codec::to_miniscript::to_miniscript_descriptor(&d, 0)` — the COMPLETE faithful watch-only descriptor (expands all keys + walks the whole tree) — then DISCARDS it: `:841 template_from_descriptor(&ms0)` (`wallet_export/mod.rs:283 Wsh(_) => WshMulti|WshSortedMulti`, top-level-wrapper-only) collapses it, and `:882 build_descriptor_string(template, &slots, k, …)` rebuilds a plain `wsh(multi(k, all-keys))`. The fix is to USE `ms0` for general policies instead of discarding it.

## The 3 caveats of raw `ms0` (all dissolved by ONE `translate_pk` pass)
`to_miniscript_descriptor(&d, 0)` returns a descriptor that is faithful in STRUCTURE but, as-is:
1. **single-path** (`/0/*`, not canonical multipath `<0;1>/*`);
2. **depth-0 keys** — md1 stores only `[chain_code‖pubkey]`, so keys serialize as depth-0 `xpub661…` ≠ the original account-depth xpub. **Byte-equality with `export-wallet --descriptor` is therefore impossible by design** (v0.49.1 impl-review I2 hit this exact point). Addresses are UNAFFECTED (derivation uses only chain_code+pubkey);
3. **network hardcoded `Main`** (md-codec `derive.rs`), so testnet would mis-render `xpub`.

Because the workspace `[patch.crates-io]` (Cargo.toml:16-17) unifies md-codec and the toolkit onto ONE miniscript (rev `95fdd1c`), `ms0` IS the toolkit's `MsDescriptor<DescriptorPublicKey>` — no string re-parse bridge. A single `translate_pk` with a `Translator` (and `translate_hash_clone!` for the hash-fragment identities) maps each key. **The Translator must be TOTAL (R0-r1 M6):** `pk()` matches `DescriptorPublicKey::XPub{…}` and returns a typed internal error (NOT panic) for `Single`/`MultiXPub` (unreachable in the non-taproot arm — `build_descriptor_public_key` only emits `XPub` — but must be total). Per-key mapping:
- `use_site_path.multipath == Some(alts)` → `MultiXPub{origin, xkey:(network-corrected), derivation_paths: DerivPaths::new(alts), wildcard: Unhardened}` — canonical `<0;1>/*` (recon-proven byte-exact incl. checksum: `…/<0;1>/*…)#v650sx0l`).
- **`use_site_path.multipath == None` (R0-r1 I3, constructible — wildcard-only keys via `bundle --descriptor "wsh(multi(2,[fp/…]xpub/*,…))"`)** → pass the `XPub` through network-corrected ONLY; keep its single-path/wildcard form, do NOT promote to `MultiXPub` or fabricate `<0;1>` (that infidelity is exactly what we're fixing). `ms0`'s per-key path is already empty here (`to_miniscript.rs:116-130`).
`d.use_site_path.multipath` is pub (`use_site_path.rs:51`).

## PART 1.A — the faithful reconstruction helper
New `fn faithful_multisig_descriptor(d: &md_codec::Descriptor, network: CliNetwork) -> Result<String, ToolkitError>`:
- `to_miniscript_descriptor(d, 0)` → map `to_miniscript` errors (the `pk(@N)`/`pkh(@N)` double-Check shape, PART 2) to a CLEAR `ToolkitError` naming the limitation + the md-codec FOLLOWUP slug (`to-miniscript-check-pkh-double-wrap`) — "this md1 encodes a key-check fragment the current md-codec cannot yet render back; the engraved card remains a faithful backup; tracked as <slug>". NOT silent, NOT a plain-multi collapse.
- `translate_pk` (the total Translator above).
- `.to_string()` (Display appends `#csum`). The bare-print path emits this string directly (it is `Display` of a parsed `Descriptor` — already valid; R0-r1 M7: `CheckedDescriptor::new` at `restore.rs:696` re-validates only on the `--format` path, which the general arm also passes through).

## PART 1.B — the structural discriminator (preserve existing paths byte-for-byte)
New `fn plain_template_from_tree(node: &md_codec::tree::Node, use_site: &md_codec::use_site_path::UseSitePath) -> Option<CliTemplate>` returning `Some` ONLY for the strictly-plain shape:
- `(Tag::Wsh, [child])` where child is `(Tag::Multi|Tag::SortedMulti, MultiKeys{indices})`, OR the `Sh(Wsh(multi|sortedmulti))` nesting;
- `indices == [0,1,…,n-1]` identity. **(R0-r1 M3: a *permuted* first-occurrence is unconstructible — md-codec `decode.rs:56 validate_placeholder_usage` rejects out-of-order first occurrences — so the only constructible non-identity plain shape is duplicate-bearing, e.g. `multi(2,@0,@1,@0)`. That MUST route to the faithful arm: `build_descriptor_string` would silently drop the duplicate.)**
- `use_site` is the standard `<0;1>` (a non-standard/`None` use-site routes to faithful, fixing that latent infidelity too).
Do NOT discriminate via `template_from_descriptor(&ms0)` — its `Wsh(_) => WshMulti` arm IS the collapse being fixed.

**Legacy `sh(…)` (R0-r1 M2):** a bare `sh(multi)`/`sh(<general miniscript>)` md1 (constructible via `bundle --descriptor`) currently refuses LOUDLY at `template_from_descriptor`'s `ShInner::Ms` arm (`wallet_export/mod.rs:279-281`). Under the new routing the discriminator returns `None` and the faithful arm reconstructs it faithfully (`to_miniscript.rs:241-253` handles Legacy) — a behavior IMPROVEMENT, but outside the old "wsh/sh(wsh)" boundary. PIN it with a test cell (in-scope: the faithful arm is shape-agnostic).
