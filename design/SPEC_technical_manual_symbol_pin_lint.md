## 0. Problem (verified)

The technical manual cites source as `file.rs:N` line numbers; they decay every merge and **nothing gates them** (no CI on the technical-manual; `api-surface-coverage.sh` is advisory name-only). A line number is **un-gateable** (a lint can't check "line N still means what the prose claims"); a *symbol name* is gateable and drift-invariant under within-file moves — which are the observed drifts (`593→745`, `724→1078`, `831→1527`). Terminal fix = replace line numbers with **audited** symbol anchors + gate symbol existence (a file-exists-only gate would miss within-file moves).

**Complete `src/` manifest (826 citation members, manifest v3 → one unified `cycle-b-audit-queue.tsv`).** Class breakdown: 325 corroborated-clean top-level (`fastpath`), 238 other top-level, 132 method (inside `impl`), 77 comma-tail (`tail:*`), 24 file-header, 10 nested/re-export, 9 collision-risk, 8 test (inside `mod tests`), 3 unresolved. **There is NO auto-pin bucket: every one of the 826 is audited** (§3); the `fastpath` flag marks corroborated-clean refs that an agent *confirms in one line* — it is a speed hint, **not a skip-set** (R3 C1). The class hint tells each agent the likely anchor *form*. Transcripts (`transcripts/api-harvest-*.md`, ~385 refs) are **out of scope** (R2: unrendered scaffolding; harness gates `src/` only; → separate FOLLOWUP).

## 1. Convention (new; documented in `AUTHORING.md`)

A source citation is `` `<path>.rs::<anchor>` `` (single backtick token). `<path>` verbatim as authored. `<anchor>` names the **most-specific item identifying the location the prose points at**, per nesting context (R2 C2):

| location | anchor | example |
|---|---|---|
| top-level `fn`/`struct`/`enum`/`trait`/`type`/`const`/`static`/`mod`/`macro_rules!` | the item name | `synthesize.rs::synthesize_descriptor` |
| **method inside `impl T`** (or `impl Tr for T`) | `T::method` (qualify — bare method names collapse + make G2 vacuous) | `tlv.rs::Tlv::write` |
| **fn inside `#[cfg(test)] mod tests`** | `tests::fn_name` | `json_envelope.rs::tests::bundle_json_view_round_trips_every_field_of_bundle_json` |
| **`pub use` re-export block / module header / line-content ref** (prose points at a line's *text*, not an item — R2 I-area / R1 I3) | bare `` `<path>.rs` `` (no `::`) | `mk-codec/src/lib.rs` |

The `.rs::` shape is not valid Rust — in docs it means "the `<anchor>` within `<path>.rs`"; Rust *module* paths (`cmd::bundle::foo`, no `.rs`) are NOT matched by the gate. **Bare `:N` continuations** (`` `:838-1277` ``) and **comma-lists** (`bch.rs:424, 440, 478`) are resolved per-member in the audit → each becomes its own `::anchor` (deduped) or collapses to one anchor when the prose genuinely means one item.

**Colliding basenames (R2 C3, narrowed per R3 I1).** 22 basenames exist in ≥2 repos. The ambiguity only bites in **non-authoritative chapters** — those whose default repo is the catch-all `toolkit` *guess* (10/14/30s/60s). **Authoritative chapters** are about one repo and their default is trusted: 21/51→md, 22/52→mk, 23/53→ms, 41/42/54→toolkit; there a bare colliding basename stands (ch52's 43 `bch.rs` cites resolve to mk correctly — qualification would be pure churn). The gate's collision test is on the **path-suffix as written across repos** (not the bare basename): `wallet_export/mod.rs` is toolkit-unique → no collision; bare `derive.rs` matches toolkit+md → collision. Net genuine collision-risk = **5** members (`derive.rs`×4 in catch-all chapters 31/33 + 1 ambiguous glossary `error.rs`). (The recon generator `cycle-b-manifest3.py` flags **9** because it approximates collision by bare *basename* — over-flagging 4 `wallet_export/mod.rs` cites whose *suffix* is toolkit-unique; R4 M1/M2. The **gate** (P1) implements the §2 path-suffix predicate, so those 4 are NOT flagged; optionally re-run the generator with the suffix predicate so recon and gate agree.) For the 5 genuine: audit agents **resolve the repo from prose** and the author writes a **repo-qualified path** (e.g. `descriptor-mnemonic/crates/md-codec/src/derive.rs::Descriptor::derive_address`). Codec/toolkit chapters keep concise bare basenames. (R4 M3: two bare `mod.rs` cites in ch52 — `mod.rs:27-31`, `mod.rs:29-37` — are path-*incomplete* (`unresolved`), not collisions; the audit supplies the parent dir, e.g. `string_layer/mod.rs`, so G2 resolves.)

## 2. Gate (new, blocking) — `docs/technical-manual/tests/symbol-ref-check.py`

Python checker (consistent with `api-surface-coverage.sh`), invoked from `tests/lint.sh` as a **blocking** step (new 7/7), in the Makefile `lint` help. Scope: `src/**/*.md` only. Two assertions:

- **G1 — line-ref ban (anti-regression):** FAIL on any source-line-ref form: (a) `\.rs:[0-9]` (file.rs:N / ranges / comma-lists), (b) a backtick token matching `^:[0-9]+(-[0-9]+)?$` (bare-`:N` continuations). **Scans the whole file text including ```mermaid```/code fences** (R0 R1 5b: skipping fences would un-gate the mermaid node where a known drift lives). Escape hatch for a *legitimate* future literal (e.g. a quoted rustc panic trace): an explicit `<!-- lint-allow-lineref -->` marker on the preceding line, documented in AUTHORING. Phase 1 verifies current `src/` is false-positive-free (R0 R1 verified no `.rs:N` in code fences; Phase 1 also checks the bare-`:N` pattern surface).
- **G2 — symbol existence (multi-segment):** for each `` `<path>.rs::<anchor>` `` token, resolve `<path>` (chapter→repo default + global basename index over the 4 codec + 3 CLI crates `src/`+`tests/`, shallowest-relpath tiebreak — the resolver logic, recon-validated) then assert **every `::`-separated segment** of `<anchor>` exists as a whole word in the file (`grep -wqF` per segment). So `tlv.rs::Tlv::write` requires both `Tlv` and `write` present (a bare `write` would pass vacuously — R2 C2; the qualifier restores discrimination). Unresolvable file OR any missing segment → FAIL. **Colliding-basename resolution (R2 C3, narrowed R3 I1):** the gate flags a collision only when the cited **path-suffix** matches files in ≥2 repos AND the chapter is **non-authoritative** (default repo is the `toolkit` catch-all guess, i.e. NOT one of 21/22/23/41/42/51/52/53/54). In an authoritative chapter, the chapter default resolves the bare basename and is accepted. For a flagged collision the gate requires the token to be **repo-qualified** — defined precisely (R3 M2) as: the path contains a `crates/<codec>/` segment OR a leading repo-dir segment (`descriptor-mnemonic/`, `mnemonic-key/`, `mnemonic-secret/`, `mnemonic-toolkit/`); a bare `src/`-prefix does NOT qualify (it doesn't name a repo). Unqualified flagged collision → FAIL "ambiguous basename; qualify the path". **Graceful degradation:** absent sibling-repo source (bare CI) → skip those refs with a warning (mirrors `api-surface-coverage.sh`); only resolvable-but-missing FAILs. (Existence is a *backstop* against hallucinated/renamed symbols; it does NOT prove the audit picked the right symbol — that is §3's job.)

Bare `` `file.rs` `` prose mentions are NOT gated (R0 I-area: 112 distinct exist incl. `bg002h/…` and 4 manual-local `examples/examples/*.rs`; gating them false-positives).

## 3. Content audit (the migration — AUDIT EVERY citation; no auto-pin bucket)

**A blind mechanical line→symbol sweep is prohibited** (R1 C1 / R2 C1: it cements drifted refs as wrong-but-existing symbols G2 cannot catch). **And there is no "trust-without-review" auto-pin bucket** (user directive; R2 C1 showed test-fns/re-exports leak into any heuristic "safe" set, and ≥6 error classes mean sampling can miss one). **Every citation member (~826) is audited.** Corroborated clean refs are a *fast-path within* the audit (confirm in one line), not a skip.

**(3a) Fan-out audit workflow.** The audit input is the **single unified `cycle-b-audit-queue.tsv`** (manifest v3; all **826** members, no autopin split — R3 C1; the old `cycle-b-autopin.tsv` is retired/removed). Per-file agents; shard the large chapters (`54`=207, `51`=185, `52`=145 members) into ≤~50-ref shards → ~22-26 agents. Each agent receives: the chapter file, ALL its queue rows (manual line + raw token + class hint `{toplevel[/fastpath]|method|test|nested|reexport|collision|tail:*|file_header}`), and the workspace source roots. The `fastpath` flag means "corroborated clean — confirm in one line", NOT skip. **Task framing (advisor — prevents lazy false-keeps):** for each ref, *locate in source what the surrounding prose is pointing at*, cite `file:line` evidence, then emit the anchor in the §1 nesting form (`name` / `T::method` / `tests::fn` / bare-file). **Repo from prose, not the toolkit-default guess** (R2 C3). Keep-vs-fix is informational (whether the line moved); the deliverable is the correct anchor regardless. Output rows: `{manual_line, old_token, evidence_file:line, anchor, one-line justification}`.

**(3b) Verification pass.** A second independent agent (or shard-swap) re-locates the evidence for **every ref whose anchor differs from the resolver's enclosing-symbol guess** (the genuine fixes/drift — where error concentrates) **plus** a structured sweep that the *applied* result is internally consistent: no two distinct prose facts collapse to one anchor unless they genuinely share one item. G2 (multi-segment existence) is the format/hallucination backstop on top.

**(3c) Apply + verify.** Apply all audited anchors mechanically (a generated edit list keyed by `(file, manual_line, old_token)`), then `make -C docs/technical-manual lint` (G1+G2+figures-cache-verify) GREEN; `git diff` review confirms only citation tokens changed (no prose/code touched).

**Mermaid special-case** (`41-bundle-anatomy.md` node `synthesize_unified<br/>(synthesize.rs:593)`): line 593 is a comment inside `synthesize_multisig_watch_only` (`synthesize.rs:488`); the node *label's* intent is `synthesize_unified` (`synthesize.rs:744`). (Corrected boundaries per R2 M1: `synthesize_descriptor` 228-**322**, `synthesize_multisig_full` 343, `synthesize_multisig_watch_only` 488, `synthesize_unified` 744.) Fix by intent → drop the parenthetical to `synthesize_unified` (label names the symbol; G1 passes, no `:N`). Editing the block rehashes the cached figure → `make figures-cache` regen. **Empirically settled (R2 I2):** bare `mmdc --input … --output …pdf --quiet` renders here (exit 0); `make figures-cache` runs idempotently (rendered=0/9 cache hits, exit 0); `figures-cache-verify` keys on the block's source-text SHA, so regen emits the new-SHA PDF + drops the orphan.
