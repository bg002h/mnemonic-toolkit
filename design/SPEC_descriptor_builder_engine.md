## 0. The architecture (3 layers — IR is the contract)

```
JSON node-tree spec  ──▶  PolicyNode IR (fragment tree)  ──▶  render(PolicyNode→String) → wsh(M) →
  (--spec <FILE|->)        (validated, versioned)              from_str(&s)?.to_string()  (canonicalize + BIP-380 checksum) →
                                                               { descriptor, bip388, cost-preview, diagnostics }
```

The engine is a **deterministic renderer + validator**, NOT a compiler (`miniscript/compiler` stays OFF). It does not optimize or auto-type-infer — it renders the authored fragment tree to a byte-exact miniscript string and validates it. (Optimization/lowering an abstract policy is the deferred general-compiler path, hosted in `md` per `miniscript-compiler-optimize-policy`.) Tractability for end-users comes from Release-B presets + the GUI, not from the engine hiding miniscript.

## 1. The node-tree IR + versioned JSON spec

`PolicyNode` is a fragment-level tree mirroring the miniscript AST (Option F). Serialized as **versioned JSON** (`{"schema_version": 1, "wrapper": "wsh", "root": <node>}`), `deny_unknown_fields`. **Encoding decision (Phase 1):** each `PolicyNode` is an **externally-tagged** serde enum (`{"<frag>": <payload>}` — exactly one key), and `deny_unknown_fields` is enforced on every struct payload. This is forced by serde: `deny_unknown_fields` is **incompatible with `#[serde(flatten)]` and with internally/adjacently-tagged enums**, so the brainstorm's "any node + adjacent `"w"` field" cannot be encoded while still rejecting typo'd fields (the funds-safety property). The wrapper is therefore its **own node** `{"wrap": {"w": "<wrappers>", "sub": <node>}}` rather than an adjacent field — same expressiveness, serde-clean, and it mirrors miniscript's own structure (`v:X` *is* a node wrapping `X`). v1 fragment set (the converged set — sufficient for all 5 archetypes incl. hashlocks):

| Node (JSON) | miniscript | Notes |
|---|---|---|
| `{"pk": "<key>"}` | `pk(<key>)` | `<key>` = a concrete `[fp/path]xpub` (multipath `/<0;1>/*` appended by the engine) |
| `{"pkh": "<key>"}` | `pkh(<key>)` | hash form (used under `v:` in inheritance shapes) |
| `{"multi": {"k": K, "keys": [..]}}` | `multi(K, …)` | wsh unsorted |
| `{"sortedmulti": {"k": K, "keys": [..]}}` | `sortedmulti(K, …)` | wsh sorted |
| `{"older": N}` / `{"after": N}` | `older(N)` / `after(N)` | relative / absolute timelock |
| `{"sha256": "<hex>"}` (+ `hash160`/`hash256`/`ripemd160`) | `sha256(<hex>)` … | hashlock (32- or 20-byte hex per fn) |
| `{"and_v": [A, B]}` / `{"or_d": [A, B]}` / `{"or_i": [A, B]}` / `{"or_b": [A, B]}` | `and_v(A,B)` … | binary combinators (fixed 2-arity arrays — serde enforces) |
| `{"andor": [A, B, C]}` | `andor(A,B,C)` | `(A and B) or C` (fixed 3-arity) |
| `{"thresh": {"k": K, "subs": [..]}}` | `thresh(K, …)` | k-of-n over sub-policies |
| `{"wrap": {"w": "<wrappers>", "sub": <node>}}` | `<w>:<frag>` | explicit miniscript wrappers (`v`,`s`,`c`,`d`,`j`,`n`,`a`,`t`,`l`,`u`) — the `w` string is emitted verbatim as a prefix on the rendered `sub` |

**No auto-wrapping / type-inference (Release A) — RULED: strictly explicit (I1).** The author (preset producer / GUI / power-user) supplies wrappers explicitly via `"w"`; a mis-typed tree is REJECTED loudly by the validation gate (§3) with a node-addressed diagnostic — not silently "fixed." This keeps the engine a **pure deterministic renderer** (matching §0's philosophy). **R0 decision (I1):** Release A owns NO safe-wrap helper; wrapper-inference (`and_v`-needs-V, `or_b`-needs-Wdu, …) is assigned to the producer layer (Release-B presets) + the GUI cycle. The existing real-vault corpus already authors every shape with explicit `v:` (`cli_non_canonical_descriptor.rs`, `cli_standalone_bijections.rs`: `and_v(v:pkh(@1),older(4032))`, `and_v(v:pk(@0),after(..))`), so explicit wrappers are demonstrably authorable and the gate's loud rejection is the backstop. **Release A is therefore the validated emit substrate the GUI's wrapper-inference layer targets — NOT itself the wrapper-inference capability** (the brainstorm's "structured-composition capability the GUI needs" is softened to this).

**Render** (`PolicyNode → String`): a straightforward recursive `Display` (each node → its miniscript text; `keys` get the `/<0;1>/*` multipath suffix; a node's `"w"` prefixes its rendered fragment). Deterministic and byte-stable. This is the engine's own emitter (§4) — NOT `build_descriptor_string`.

## 2. The subcommand surface

`mnemonic build-descriptor` (name — R0 to confirm vs `descriptor`/`policy`):
- `--spec <FILE|->` — the JSON node-tree spec (file or stdin). Canonical input.
- `--network <NET>` — required (taproot/segwit address rendering + descriptor network).
- `--format <FMT>` — output: `descriptor` (default; bare concrete `wsh(M)#csum`) or `bip388` (wallet-policy JSON). **R0 decision: v1 ships exactly these two.** The export-wallet emitter set (`bitcoin-core`, …) is template-coupled (it routes through `build_descriptor_string`'s hardcoded `CliTemplate` match — see C1/§4) and is deferred to keep `wallet_export`'s template coupling out of Release A.
- `--json` — structured envelope (descriptor + bip388 + cost-preview + diagnostics) for the GUI; default human text shows the descriptor + a branch summary + the cost table.
- `--spec-schema` — dump the versioned node-tree JSON-schema (node grammar + value constraints) instead of building; the GUI consumes this. **R0 decision: a `--spec-schema` flag on `build-descriptor`** (dump-and-exit), not a separate subcommand — it is spec-coupled to this command.

## 3. The validation gate (the funds-safety core — substrate exists)

Emit is **gated**, in order; each failure → a **node-addressed structured diagnostic** (the JSON path to the offending `PolicyNode`), never an opaque string:
1. **Deserialize + schema-validate** the JSON (`deny_unknown_fields`, version check, per-node field/arity/hex/threshold checks: `1 ≤ k ≤ n`, hashlock hex length, timelock `1 ≤ N < 2^31`). Node path is read directly from the serde path.
2. **Render → `wsh(M)` → `MsDescriptor::<DescriptorPublicKey>::from_str`** (the miniscript **type-check ONLY**; the dual-context parse pattern is `cost/translate.rs:81-83`). **Empirically characterized (F1):** `Descriptor::from_str` rejects type/structural errors (e.g. a missing `v:` wrapper → "cannot accept children of types B and B") but is **LENIENT on the funds-footgun sanity rules** — it returns OK for sigless / mixed-timelock / repeated-key policies. So step 2 catches *only* type errors; the footguns are step 3's job (the two gates are genuinely distinct — do NOT collapse them, and never treat `from_str`-OK as a safety verdict). A type failure is localized by the §3.4 per-subtree walk → the minimal failing subtree's node.
3. **Funds-footgun rules** (the BIP-built-in sanity rules; available under `std`, NOT `compiler`). **The oracle is explicit `Miniscript::sanity_check()` (`analyzable.rs:225`), NEVER `from_str`-OK (F1).** Whole-tree `sanity_check()` is the **fast all-clear gate**; on failure it returns only a single payload-less first-failure variant in fixed priority (Sigless → Malleable → ResourceLimits → RepeatedPubkeys → HeightTimelock, `analyzable.rs:133-146`) with **no sub-location** — so **localization is NOT read off `sanity_check`**; it is computed by the §3.4 per-subtree predicate re-check. The mapped rules:
   - `SiglessBranch` (anyone-can-spend path), `Malleable`, `BranchExceedResouceLimits` *(sic — the upstream identifier is misspelled, missing an 'r'; `analyzable.rs:139`)*, `RepeatedPubkeys`, `HeightTimelockCombination` (mixed height/time unspendable-path — *the* "wrong timelock loses money" guard).
   - The 6th variant `ContainsRawPkh` (`analyzable.rs:145`) is **unreachable for builder-emitted trees** (the IR has no raw-`pkh`-without-key node) → not mapped; asserted unreachable.
   - **No silent opt-out, and Release A ships NO `--allow` flag** (C3 — `sanity_check` is param-less, so there is no in-place opt-out substrate; the `ExtParams`/`from_str_ext` path is deferred, §3.5). The reviewed escape hatch for a deliberately "insane" policy is the existing raw `--descriptor` intake door. **R0-verified empirically (F3, oracle = explicit `sanity_check()`): all 5 *final* (distinct-key) archetypes PASS** (incl. the decaying-multisig combining `older` + `after`). Cutting `--allow` DOES block one shape — a *same-key* degrading threshold trips `RepeatedPubkeys` (F2) — but that is correct (miniscript deems same-key degradation unsafe); such a user takes the raw `--descriptor` door (or the deferred `--allow`).
4. **Per-branch satisfiability** via `plan()` (reuse `cost/enumerate.rs:258` `desc.plan(&a)` over the asset powerset, after the §4 multipath→single-path projection) — flag any branch with no satisfying configuration (a dead branch = locked funds).
5. **Build-time complexity cap (NET-NEW — the "always-previewable envelope"):** bound tree depth / leaf-count / distinct key+hash count at CONSTRUCTION so the cost preview ALWAYS renders. `cost/enumerate.rs` (`checked_pow` guard `:113-115`, `raw > hard_cap` refusal `:119-120`) REFUSES enumeration past `--max-conditions` (`ConditionsTooMany`) — which would make the review surface vanish exactly when a policy is most complex/dangerous. Move the cap UPSTREAM: refuse to BUILD a tree whose `2^(keys+hashes) × timelock-states` would exceed the preview cap.
   - **Key count MUST use enumerate's dedup** — enumeration dedups keys via `BTreeSet<DescriptorPublicKey>` (`enumerate.rs:374-391`, over `for_each_key` of the parsed descriptor); the pre-render cap must dedup keys identically or the cap and the actual enumeration disagree on `n_keys` (architect caveat). **Dedup on the full rendered key-expression** (origin + xpub + `/<0;1>/*` suffix — the translated `DescriptorPublicKey`), NOT the IR's bare `<key>` string: the same xpub at two different origins/paths is two distinct enumeration keys. A Phase-3 cap test with one xpub under two origins discriminates this (R0-r2 M1).
   - Timelock-state count classifies each `older(N)`/`after(N)` exactly as `enumerate.rs` does (abs-height vs MTP against `500_000_000`; rel-blocks vs 512s against `TIME_LOCK_FLAG`) → `n_tl_states = n_abs × n_rel`, each ∈ `1..=3`.
   - Past-envelope policies → refuse with a pointer to the raw `--descriptor` door.

Only after all 5 pass does the engine emit.

### 3.4 Node-addressed localization (the keystone — the per-subtree mechanism)
The IR carries each node's JSON path. To localize a step-2 or step-3 failure, walk IR subtrees **deepest-first**; for each subtree render it to its miniscript string and attempt `Miniscript::<DescriptorPublicKey, Segwitv0>::from_str_ext(s, &ExtParams::insane())`:
- **Step 2 (type/parse):** the minimal subtree that fails `from_str_ext` is the offending node. (`ExtParams::insane()` relaxes only the 5 sanity rules — it does **NOT** relax the `Base::B` typecheck, F1 — so a type error still fails the insane parse and is cleanly separable from a footgun.)
- **Step 3 (funds-footgun):** on each subtree that parses, run the matching public predicate — `requires_sig` (`analyzable.rs:187`, SiglessBranch), `is_non_malleable` (`:190`, Malleable), `within_resource_limits` (`:195`), `has_repeated_keys` (`:201`, RepeatedPubkeys), `has_mixed_timelocks` (`:198`, HeightTimelock). Each is a **local property of the subtree** (verified — all are `pub fn(&self) -> bool` on `Miniscript`), so the standalone verdict equals the embedded contribution; the minimal (deepest) failing subtree is the diagnostic's node. Inherently cross-branch rules (mixed-timelock, repeated-keys) resolve to the **nearest common ancestor** — the minimal subtree containing the conflicting pair.
- **B-type restriction (R0-verified caveat):** `from_str_ext` rejects a non-`B` top-level fragment with `Error::NonTopLevel` (it enforces `ty.corr.base == Base::B`, `mod.rs:850-853`). A non-B subtree (e.g. an explicitly `v:`-wrapped child, type V) is **not standalone-testable** → the walk skips it and localization lands on the **nearest B-typed ancestor**. This is the precision floor (a `v:pk` defect surfaces as "the `and_v` subtree at path X"), not a soundness gap — the failure is still reported, with node-level (not always leaf-level) granularity.
- Trees are cap-bounded (step 5) → the `O(n²)` walk is free.

### 3.5 Deferred `--allow` (FOLLOWUP, not Release A)
A future reviewed opt-out maps each `--allow <variant>` to the matching `ExtParams` toggle (`top_unsafe` / `malleability` / `resource_limitations` / `repeated_pk` / `timelock_mixing`; `analyzable.rs:28-121`) applied at the **parse stage** via `from_str_ext` (NOT at `sanity_check`, which takes no params). Deferred from Release A; file as a FOLLOWUP. Until then, raw `--descriptor` is the documented escape hatch.

## 4. Emit (review-before-emit bundle)
- **Concrete descriptor** `wsh(M)#csum`: the engine **renders `PolicyNode → String` itself** (§1's recursive `Display`), then canonicalizes + checksums via the round-trip idiom `MsDescriptor::from_str(&rendered)?.to_string()` — the 2-line idiom `build_descriptor_string` uses internally (`wallet_export/pipeline.rs:28-30`). **NOT via `build_descriptor_string` itself (C1):** that function is a hardcoded `match template { Bip44 => "pkh(..)", WshMulti => "wsh(multi(..))", … }` over the fixed `CliTemplate` enum (`pipeline.rs:86-104`), has no recursion, and cannot render `wsh(andor(...))`. (Pipeline-emit reuse is thus ~2 lines, not the function — corrects the recon's "~60% reuse incl. `wallet_export/pipeline.rs`" sizing.)
- **BIP-388 wallet-policy JSON** via `descriptor_to_bip388_wallet_policy` (`pipeline.rs:166`) — this one **IS** shape-general (string-based `iter_pk` + longest-first `replacen`, `:183-204`; multipath guard `:171`; `/<0;1>/*` suffix `:218`) and works on any parsed `wsh(...)`. Round-trips through the v0.49.0 `--descriptor` policy intake.
- **`compare-cost` per-condition preview** (reuse `cost/enumerate.rs` — keeps BOTH wsh + tr columns even though emit is wsh-only) + a human branch summary.
  - **Multipath projection (NET-NEW glue, I2):** the cost intake `strip::translate_descriptor` (`cost/strip.rs:20-31`) calls `derive_at_index(0)`, which **errors on multipath** descriptors (miniscript `descriptor/mod.rs:706`; doc-condition at `:705`). The builder emits `/<0;1>/*` multipath, so the preview must first split via `into_single_descriptors()` (`mod.rs:946`) and enumerate over the `[0]` (external) descriptor — cost is path-invariant — before `translate_descriptor`/`plan()`. The existing wildcard compare-cost test (`cli_compare_cost.rs:768`) uses single-path `/0/*`, so this multipath path is untested/net-new.
- `--json`: `{ descriptor, bip388, cost: [...], diagnostics: [] }`; on failure: `{ diagnostics: [{node_path, kind, message}] }` (exit 2).

## 6. Release-A R0 acceptance fixtures (the keystone — hand-authored, NOT preset code)
Author the 5 archetypes as **node-tree JSON fixtures** (in `tests/fixtures/descriptor_builder/`). All shapes below PASS explicit `sanity_check()` against distinct keys (F3), and together exercise the full v1 fragment set:
- `simple-timelocked-inheritance` — `wsh(or_d(pk(A),and_v(v:pkh(B),older(N))))` *(pk, pkh, older, and_v, or_d, wrap)*.
- `decaying-multisig` — `wsh(andor(multi,older,andor(multi,older,and_v(v:pk,after))))` *(multi, older, after, andor, and_v, wrap)*. **Note the `v:` on the `pk` under `and_v`** (corrected from the brainstorm's `and_v(pk,after)`, which is mistyped: `and_v` needs a V-typed left arg, `pk(K)` is type B → the `v:` wrapper is mandatory). `older` + `after` do not trip `HeightTimelockCombination`.
- `kofn-recovery` — `wsh(or_d(multi(k,…),and_v(v:pk(R),older(N))))` *(multi, pk, older, or_d, and_v, wrap)*.
- `tiered-recovery` — `wsh(or_i(sortedmulti(2,…),and_v(v:older(N),thresh(2,pk,s:pk,s:pk))))` *(sortedmulti, or_i, thresh, older, and_v, wrap)*. **Replaces the brainstorm's "degrading-threshold"** — a *same-key* degrading threshold (`or_d(multi(n,K…),and_v(v:multi(m,K…),older))`) reuses keys → `RepeatedPubkeys` → rejected (F2); a true same-key degradation is an "insane" miniscript only reachable via the deferred `--allow`/raw `--descriptor`. Tiered-recovery is the sane distinct-key analogue (primary quorum OR a timelocked recovery quorum) and additionally covers `sortedmulti`/`or_i`/`thresh`.
- `hashlock-gated` — `wsh(andor(pk(A),sha256(H),and_v(v:pk(B),older(N))))` *(pk, sha256, andor, and_v, older, wrap)*.

(`or_b` and the `hash160`/`hash256`/`ripemd160` leaves are not in an archetype; they are covered by the §7 IR/render unit cells + a Phase-2 targeted `or_b` gate cell.)

These (a) prove the v1 fragment set is expressive enough; (b) freeze the schema against reality; (c) **pre-pin the canonical descriptor strings** Release B's producers must reproduce.

**Phasing (per §9):** author fixture **inputs** in Phase 1 (proves IR expressiveness + freezes the schema — does NOT need emit); pin the descriptor + bip388 **goldens** in Phase 3 (goldens require emit, which Phase 3 delivers — "authored" ≠ "golden-pinned"). Capture goldens only from a verified-correct run.

## 7. Test plan (per-phase TDD; RED before impl)
- **IR/render unit cells:** each `PolicyNode` variant → exact miniscript text; multipath suffix; wrapper prefix; `deny_unknown_fields` (unknown top-level field, typo'd struct-payload field, AND a sibling key on a leaf node — the externally-tagged single-key rule); version mismatch refused; **wrong-arity rejection** for the fixed-array combinators (`{"and_v":[A]}` / `[A,B,C]`, `{"andor":[A,B]}` all refused — serde `[T;N]` enforces); **enum-coverage cross-check** (`node_kinds_cover_enum` asserts `all_variant_samples`' tag-set == `NODE_KINDS`; `grammar_matches_node_kinds_hand_list` asserts `NODE_KINDS` == `NODE_GRAMMAR` — together catching *drift* between the three hand-lists; neither catches a variant jointly omitted from all three — enum→list is author discipline; an airtight guard would need a variant-enumerator macro, out of scope; risk #2 partial freeze, R0-r3/r4 I1).
- **Validation gate cells (each must RED a bad tree, GREEN a good one — oracle is the gate's verdict, NEVER `from_str`-OK, F1):** type-error (missing `v:`) → node-addressed diagnostic via §3.4; `sanity_check` rejections (sigless / malleable / mixed-timelock / resource-limit / repeated-key) each → the right node diagnostic (note: `from_str` is LENIENT on all four footguns — the cell must assert the *step-3* verdict); dead-branch (`plan` empty) → flagged; over-envelope tree → refused at build (not at enumerate); B-type restriction case (defect under a `v:` child localizes to the nearest B ancestor); a `repeated-key` cell specifically guards that step 2 passes but step 3 rejects.

- **Round-trip:** built descriptor → `export-wallet --format bip388` reproduces the bip388; → `--descriptor` re-ingests (the v0.49.0 intake).
- **`--spec-schema`:** matches the serde structs (self-test); the node grammar is complete for the 5 fixtures.
- **Negative discrimination:** mutate a fixture's threshold/timelock → the golden FAILS (non-vacuous).

## 10. The 3 risks R0 must nail (from the brainstorm)
1. **Canonical-form stability:** golden-pin each archetype's byte-exact `Display`-canonicalized descriptor + bip388 round-trip; the residual drift surface is miniscript's `Display` (rare, CI-caught).
2. **The node-tree schema is a NEW versioned contract** separate from the flat `gui-schema` flag projection; freeze + gate it in Release A before the GUI/Release-B pin it.
3. **Structured per-node diagnostics** via the §3.4 per-subtree re-check (NOT read off `sanity_check`, which yields only a payload-less first-failure with no location) — the difference between "validates" and "guides"; the GUI contract depends on it. Precision floor: non-B subtrees localize to the nearest B-typed ancestor (§3.4 B-type restriction).

### Out of scope (Release A)
Archetype preset producers + flag shorthand (Release B); the GUI wizard (separate cross-repo cycle); `tr(...)` taptree output (wrapper-strategy seam); the optimizing compiler / abstract-policy lowering (lives in `md`); auto-wrapping/type-inference; the `--allow`/`ExtParams` reviewed opt-out (§3.5 FOLLOWUP); arbitrary past-envelope policies (→ `--descriptor`); any signing.
