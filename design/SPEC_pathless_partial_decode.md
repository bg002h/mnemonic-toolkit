## Guiding principle (standing)
Maximally expressive OUTPUT, permissive INPUT — BOUNDED by **never silently misrepresent** (permissive ≠ silently-wrong-wallet). [[feedback_guiding_principle_expressive_output_permissive_input]]

## Problem
`md encode` exits 0 and mints an md1 card for every `canonical_origin == None` shape with no `--path` (tr+tree, `sh(sortedmulti)`, `sh(multi)`, bare `wsh`, raw miniscript). `md decode` then HARD-REJECTS `MissingExplicitOrigin` (exit 1). An un-readable engraved card is a funds trap. The template itself is fully on the card — only the derivation ORIGIN is elided-and-unresolvable (no canonical default for the shape).

## Reframe (architect-verified)
- A dead card and a decodable template are byte-identical except the tree SHAPE; both carry `PathDeclPaths::Shared(OriginPath{components:[]})`. `canonical_origin(tree)` (`canonical_origin.rs:46-85`) returns `Some` (→ decodes) or `None` (→ rejects).
- At the `MissingExplicitOrigin` reject (`validate.rs:221-246`, called from `decode_payload` `decode.rs:75`), the full tree/`@N`/use-site are already decoded; `descriptor_to_template` (`render.rs:52-62`) would render the card fine.
- The RESOLVED (canonical-filled) origin is never displayed; the RAW elided `path_decl` IS emitted in `--json` (`format_origin_path`, `json.rs:222-263` → `"path_decl":{"data":"m","tag":"Shared"}`).

⇒ "partial-decode-with-placeholders" = render the always-renderable template + a text `origin: «unspecified»` marker (partial only) + exit 4; no missing keys, no literal token.

## Architecture — per-call-site, NOT a global gate-relax (architect C1)
Do NOT globally relax `decode_payload`. Introduce a partial-allowing decode entry point that the render-only commands opt into; all origin-consuming/verifying commands keep strict behavior.

**md-codec new surface:**
1. `Descriptor::unresolved_origin_indices(&self) -> Vec<u8>` — the set of `@N` whose origin is empty AND `canonical_origin(tree)` is `None` (idx is `u8` throughout — `Descriptor.n:u8`, `MissingExplicitOrigin{idx:u8}` — architect M-2). Built on **`validate_explicit_origin_required` SEMANTICS** (`validate.rs:221-246`), NOT `expand_per_at_n`'s (they diverge — architect I1). Returns `[]` on any card that decodes cleanly today (canonical shape, or explicit origin present).
2. **Partial-allowing decode threaded through ALL THREE decode layers the consumers actually call (architect C-1 — NOT just `decode_payload`):** `decode_payload`, `decode_md1_string` (which routes chunk-forms into `reassemble`, `decode.rs:101-103`), AND `reassemble` (`chunk.rs`) each gain a partial variant/param (e.g. `…_opts{allow_unresolved_origin: bool}`). Default behavior is UNCHANGED (still rejects) — every strict caller keeps the gate unless it explicitly opts in. `decode_with_correction` (repair) is NOT given a partial variant (repair stays strict — see per-command table).
   - **INVARIANT (verbatim, funds-load-bearing):** partial mode relaxes ONLY the `validate_explicit_origin_required` OUTCOME (records unresolved indices instead of erroring). It does NOT relax any other decode check: per-chunk BCH, chunk-header consistency (`chunk.rs:348-361`), index-gap (`:363-372`), and **the derived-chunk-set-id / content-id check (`chunk.rs:383-391`) REMAIN ENFORCED** — a chunked dead card is partial-rendered only if its content-id reassembly PASSES; a doctored-content-id chunked card still REJECTS under partial decode. (The naive "catch `MissingExplicitOrigin`, re-decode concatenated bytes directly" shortcut is FORBIDDEN — it drops the content-id oracle, the exact aliasing class v0.86.0 + `prop_repair_never_wrong.rs:439-483` guard.) Feasible: `compute_md1_encoding_id`→`encode_payload` runs no origin gate (`encode.rs:100-110`), so the content-id check works on a partial descriptor.
3. `EmptyTlvEntry`-class reject (architect I1): an empty `OriginPathOverrides[idx]` entry must be rejected consistently by BOTH the decode gate and `expand_per_at_n` (`canonicalize.rs:465` currently skips its reject when an empty override is present → silently returns origin `[]`; probe-proven policy-id divergence `e54fed29…` vs `eb6d5e10…`). Add the reject; it **stays FATAL even in partial mode** (an empty-override wire is MALFORMED, not a dead card — it must not partial-render). Not a breaking change (no encoder emits `origin_path_overrides`; md-cli maps origins into `path_decl` `template.rs:500-506`; all committed vectors carry `origin_path_overrides: null`; existing `EmptyTlvEntry` covers only empty *vectors* `tlv.rs`, not zero-component entries). The 12 committed `MissingExplicitOrigin` pins stay green under the strict default.

| Command | Behavior | Mechanism |
|---|---|---|
| `md decode`, `md inspect` (text template) | Partial-allowing: render template + `origin: «unspecified — supply on restore»` (partial only) + stderr note + **exit 4**. `md inspect` omits the `wallet-policy-id` line on partial (keeps `md1-encoding-id` + `wallet-descriptor-template-id`, no origin dep, `identity.rs:71-96`). | opt into partial decode + `unresolved_origin_indices()` |
| toolkit `mnemonic inspect` | Same partial render (shared byte-identical renderer). exit 4 on partial. | inherits |
| `md encode` | On `canonical_origin==None` + no `--path`: loud stderr advisory (mirror F-A4 footgun) nudging `--path` for a fully-decodable backup. Still emits, **exit 0**. | encode-side check |
| `md address`, `md inspect` policy-id, toolkit `address`/`addresses`/`export-wallet` faithful, `restore --md1` | UNCHANGED — keep the strict `decode_payload` + `expand_per_at_n` fail-close (VERIFIED covered: `restore.rs:3181`, derive/`to_miniscript` paths). | no opt-in |
| toolkit `verify-bundle` (policy/descriptor flows) | **EXPLICIT partial gate (architect C1, funds-critical):** opt the supplied-card decode into partial-allowing `reassemble`, then call `unresolved_origin_indices()`; if non-empty → `result: partial` / **exit 4**, NEVER a pass (its compares exclude origin `:2903-2923`/`:3077` and it never calls `expand` on the supplied card → CANNOT be delegated). Precedence: any FAILED structural check ⇒ `mismatch` beats `partial` (a structurally-wrong dead card must not read as merely partial — M-1). | partial-allowing decode + query at the real supplied-card sites `:2450` and `:3045` (`:2846` = consumption `match` of `:2450`) |
| toolkit `verify-bundle` template route `:388` + `restore.rs:316` + advisory decoders `:3193`/`:3403` | **STRICT (architect I-1)** — a dead keyless-template card falls through `:388` strict → existing `--template is required` / policy mismatch (fail-closed, exit≠0, NO false-pass). Do NOT opt `:388` into partial (would route unresolved-origin cards into template-completion paths not designed for them). `:3193`/`:3403` stay strict-and-bail. | unchanged strict decode |
| `md repair` + toolkit repair | **STRICT — repair does NOT opt into partial decode (fold refinement, dissolves architect I-2).** Its decode oracle (`repair.rs:118`/`:1491`/`:1641`, `decode_with_correction`) stays strict → a correction resolving to a dead card is PRUNED (fail-closed, exit 2); an untouched dead card is UN-repairable (exit 2, UNCHANGED). No candidate-set enlargement, no Blessed-partial, no v0.86.0-composition. Repairing a CORRUPTED pathless card → exit-4 partial candidate is an explicit P1 FOLLOW-UP (`repair-corrupted-pathless-card-partial`), NOT built here. | no change (strict `decode_with_correction`) |
| `md verify` (md-cli standalone) | UNCHANGED — byte-compares full re-encoded payload incl. origin (`md-cli/src/cmd/verify.rs:37-45`) → already SAFE. | n/a |

**Strict consumers unchanged / fail-closed (M-4, complete map):** `md verify` (decodes at `verify.rs:18-23` BEFORE its full-payload byte-compare → SAFE via the strict decode), `md bytecode` (`bytecode.rs:10-13`), `mnemonic word-card` (`word_card_adapter.rs:103` — a legitimately-minted dead card cannot be word-carded; raw `MissingExplicitOrigin`), `xpub-search` intake (`descriptor_intake.rs:224`), `bundle` mint-side self-check/advisory (never mints dead cards — origins built fresh). None opt into partial. `convert`/`silent-payment`/`import-wallet`/`compare-cost` have NO md1-decode caller (grep-verified).

## JSON contract (additive)
- `md decode --json` / `md inspect --json` gain an additive `partial` object on the partial case: `{"reason":"missing_explicit_origin","unresolved_indices":[…]}`. COHERENT with the already-emitted raw `path_decl:"m"` (do NOT double-represent the origin — architect I2). `wallet-policy-id` omitted on partial.
- The top-level JSON is a hand-built `serde_json::Map` with conditional inserts (`decode.rs:19-31`/`inspect.rs:22-49`, M-3) — the additive `partial` key is inserted only on the partial case. `SCHEMA` stays `"md-cli/1"` (additive; no existing field changes).
- toolkit `verify-bundle --json`: `result` gains the value `"partial"` alongside the existing `"ok"`/`"mismatch"` (`schema_version:"4"`, M-1) + exit 4. **Precedence: any failed structural check ⇒ `"mismatch"` (its existing exit), NOT `"partial"`** — `partial` fires only when the SOLE deficiency is the unresolved origin. This is a `--json` wire-shape change (new enum value) → M-6 GUI check (below).

## Exit codes (architect M2 — no collision)
- `md decode`/`inspect`: full decode → exit 0 (byte-identical to today); partial → exit 4. (md-cli returns exit 4 nowhere today; toolkit exit-4 = VERIFY-ME family — consistent.)
- `md encode`: exit 0 + advisory (unchanged success).
- `verify-bundle`: partial → exit 4.
- Manual exit-code tables updated for each surface.

## Testing
- **Goldens** (before/after) for each `canonical_origin==None` shape: tr+tree, `sh(sortedmulti)`, `sh(multi)`, bare `wsh`, raw miniscript body. Partial render + exit 4 + JSON `partial`.
- **Chunked-card partial (C-1):** a MULTI-CHUNK dead card partial-renders + exit 4 (proves the partial variant threads through `reassemble`).
- **Chunked content-id oracle intact (C-1, funds-critical RED-proof):** a multi-chunk dead card with a DOCTORED chunk-set-id still REJECTS under partial decode (origin relaxation must NOT drop the content-id check).
- **BOUNDARY (RED-proof):** canonical shapes (`tr(@0)`, `wpkh(@0)`, `sh(wpkh(@0))`, `wsh(multi)`) stay exit 0, byte-identical text + JSON. Any drift fails.
- **verify-bundle NEGATIVE (funds-critical):** a keyed empty-origin `canonical_origin==None` card via BOTH `--descriptor` mode AND the `:2450` multisig policy flow → `result: partial`/exit 4, NEVER ok/exit 0 (RED-proof: without the explicit gate this false-passes). Plus `mismatch` beats `partial` (a structurally-wrong dead card → `mismatch`, not `partial`).
- **verify-bundle dead-TEMPLATE (I-1):** a dead keyless-template card (`wsh(multi(2,@0,@1))` no `--path`) via the `:388` template route → fail-closed (`--template is required`/mismatch, exit≠0), NOT a pass and NOT spuriously partial.
- **verify↔restore parity:** the same card `verify-bundle` marks partial, `restore --md1` refuses — no divergence where verify says ok.
- **repair STRICT (I-2 fold):** untouched dead card → `md repair` exit 2 (unchanged); a BCH correction resolving to a dead card → pruned, exit 2 (never Blessed/exit-5). Auto-repair does NOT fire on an intact dead card in `verify-bundle`/`inspect` (partial-decode ≠ decode-err, so the `is_err()` auto-repair trigger no longer fires — assert clean fall-through). [convert has no md1 intake — M-R2-1.]
- **I1 empty-override:** the `sh(sortedmulti(2,@0,@1))` + empty `OriginPathOverrides[0]` wire now rejected consistently (decode + expand, and FATAL in partial mode); policy-id no longer computed from an empty `@0` origin.
- **Round-trip (RED-proof, M1):** partial-decode → re-encode byte-identical (re-encode canonicalizes indices only, `encode.rs:100-102`, never fills origin).
- **Cross-binary parity:** `md decode` template == `mnemonic inspect` template on the same partial card; both exit 4.
- **toolkit `mnemonic inspect`** exit-4 + JSON `partial` asserted directly.
