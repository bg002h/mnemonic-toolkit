## T2-a (#6) — toolkit `prop_repair_never_wrong` property harness
**Gap:** the repair engine (`crates/mnemonic-toolkit/src/repair.rs`; auto-repair fires by DEFAULT) is covered only by fixed vectors for the never-wrong-payload property. proptest is already a dev-dep (`Cargo.toml:76`).
**Harness location — IN-CRATE `#[cfg(test)]` unit module, NOT a `tests/` integration test** (folded post-R0 after the #6 implementer's verified STOP+report — see `design/agent-reports/test-hardening-T2a-blocker-integration-test-visibility.md`). The toolkit is a BINARY crate: `repair`/`indel`/`error` are `pub` **only under `#[cfg(fuzzing)]`** (`lib.rs:144-190`) and plain-private in `main.rs:3-41`, so `repair_card`/`recover_indel`/`SetVerify`/`RepairError` are UNREACHABLE from an integration test (`tests/`). Add the harness as `crates/mnemonic-toolkit/src/prop_repair_never_wrong.rs`, declared `#[cfg(test)] mod prop_repair_never_wrong;` in `main.rs` — the repo's established pattern for these modules (`indel.rs:302`, `repair.rs:1910` are already `#[cfg(test)]` in-crate). This has direct private access to `crate::repair::{repair_card, SetVerify, RepairError, CardKind}` + `crate::indel::{recover_indel, IndelOracle}` INCLUDING the mock-oracle ambiguity path (CLI-unreachable: Ambiguous indel ≈2⁻⁶⁵, `cmd/repair.rs:455-459`); stays TEST-only/NO-BUMP (nothing added to the `#[cfg(fuzzing)]` lib block or shipped binary; `#[cfg(test)]` compiles only under `cargo test`); and runs direct-call (no per-case subprocess). All oracles, constructed/pinned F4 vectors, and named mutations below are UNCHANGED by this relocation.
**Fix (test):** a proptest layer for the ≤t/indel invariants PLUS deterministic constructed/pinned cells for the F4 miscorrection leg (the random rate is too rare to RED-prove — R0 measured ≈3.5e-5/trial, so proptest alone can't gate it).
- **≤t substitution leg (proptest):** random payload → encode → inject k≤4 substitutions **scoped correctly** (I4: data-part only, lowercase charset, distinct positions, new≠old; for multi-chunk cards the ≤4 budget is PER-CHUNK — else `HrpMismatch`/mixed-case reject cause spurious RED, md `chunk.rs:435-439`) → `repair` MUST emit **exactly the original** card (oracle = original bytes). **ms1: assert equality ONLY, NOT `Blessed`** (I4b — any touched ms1 correction is `Unverified` by Cycle-F design, `repair.rs:1163-1172`). RED-proof: mutate the correction path to return a wrong codeword → fails.
- **single indel leg (proptest):** one random insert/delete → `indel::recover_indel` (I4c — `recover_indel_card` is `pub(crate)`, use the public `recover_indel`) returns `Unique(original) | Ambiguous | Unrecoverable` — if `Unique`, it MUST equal the original. RED-proof (named): mutate to "return first hit without the dedup/ambiguity fold" (`indel.rs:116-121`) → RED on any ambiguous seeded case (the ambiguous-construction recipe is `recover_indel_reports_ambiguous_on_multiple_distinct_recovered`, `indel.rs:356-374`).
- **beyond-t / F4 leg — DETERMINISTIC constructed vectors (NOT random proptest):** recast onto the real repair tri-state (`Blessed` / `Unverified{reason}` / `Err(SetReassemblyMismatch)`):
  - **(a) ms1 demotion (pinned):** a touched ms1 repair (any `RepairOutcome` with non-empty `repairs`) ⇒ `Unverified`. RED deterministically under removing the Cycle-F demotion at `repair.rs:1163` (fires on EVERY touched ≤t ms1 case). This also pins the "advisory" property (ms1-only).
  - **(b) mk1 multi-chunk set-reverify (constructed):** build a doctored 2-chunk mk1 card — copy a valid 2-chunk card, alter one payload symbol in one chunk, RECOMPUTE that chunk's string-layer BCH checksum via public `bch_create_checksum_*`, then inject 1–4 substitutions so `repair` *touches* it. The 4-byte cross-chunk hash then fails set-decode ⇒ assert `Err(RepairError::SetReassemblyMismatch)`. RED under "delete `verify_mk1_set`" (`repair.rs:1115`) ⇒ `Ok(Blessed)` ⇒ the test's INDEPENDENT `mk_codec::decode(corrected)` oracle mismatches ⇒ RED, single-mutation, deterministic.
  - **(c) single-string mk1/md1 fully-valid alias — SCOPED OUT (documented residual):** single-string mk1 has NO cross-chunk hash (`mk pipeline.rs:9-11`) and md1 delegate is always-`Blessed`, so a valid-but-wrong single-string alias IS blessed today — do NOT assert it never blesses (that would RED vs master → a STOP-clause stall). Pin the CURRENT `Blessed` behavior as a **documented-residual KAT** + file FOLLOWUP `repair-single-string-fully-valid-alias-second-oracle` (cite eval F4's "payload-derived second oracle or demote" prescription).
  - **(d) random >4 injection:** keep ONLY as a never-panics / never-returns-original smoke (not a bless-property gate).

## T2-b (#7) — md `bch_exhaustive_sweep` + fix `parity_smoke` silent self-skip
**Gap 1:** md-codec BCH correction has no exhaustive small-error proof (oracle = the injected pattern, ground truth by construction).
**Fix:** add `crates/md-codec/tests/bch_exhaustive_sweep.rs` (I3 — a decode is ~259µs debug, so "all-pairs 2-error" = 4.1M decodes ≈ 17.7 min debug, NOT "seconds"): **exhaustive 1-error** (all 93 positions × 31 nonzero values = 2,883 decodes, <1s) + **all 4,278 position-pairs × a bounded seeded magnitude subset** (~8 pairs each ≈ 34k decodes ≈ 9s debug) — inject → `bch`-correct → assert recovered == original (injected pattern IS the oracle). Gate the FULL all-pairs 2-error sweep behind `#[ignore]` (release-only). **Acceptance-rate cell (I3+M1):** seeded, **≥1e5 trials** (~26s debug), inject e≥5 errors, count = **any `Some` return** (NOT "≠original" — the original never returns; but under a cap-relax mutation the decoder returns *correct* 5-error decodes, which an any-`Some` bound catches loudly while a wrong-output-only bound survives), assert below a bound **re-measured at THIS test's own (L, seed, error-mix)** (M1 — the eval's ~1.15e-4 was at a different L; ~8.5e-5 measured at L=93; do not import). Name the mutation this cell RED-proves: **relaxing the `deg > 4` / L≤4 cap at `bch_decode.rs`** (→ the cell's any-`Some`-at-e≥5 count jumps).
**Gap 2 (funds-relevant):** `crates/md-codec/tests/parity_smoke.rs` (:82-94) **silently self-skips** when the toolkit binary is absent (returns green if HOME unset or the binary not found) — so the only cross-implementation BCH check **never runs in CI**. **Fix:** replace the external-binary dependency with an in-tree **independent PGZ reference decoder**, run ALWAYS-ON (remove the silent-skip `return`s). **I2 — the reference form is load-bearing:** a brute-force ≤4-error nearest-codeword search is INFEASIBLE (≈2.7e12 residue checks/word). Use **Peterson–Gorenstein–Zierler**: syndromes computed DIRECTLY from the received word (`Σ cᵢ·(βʲ)ⁱ`, NOT from md's residue/polymod plumbing), **Gaussian-elimination** locator (deliberately NOT Berlekamp–Massey), direct root trial over the 93-orbit (not md's Chien), linear-system magnitudes (not Forney), field arithmetic over **GF(1024) = GF(32²)** (the extension `ζ² = ζ + 1` that β and the syndromes live in — `bch_decode.rs:75-80`; syndromes are `[Gf1024; 8]`) from INDEPENDENTLY-generated exp/log tables — sharing only code-DEFINITION constants (the GF(32) reduction poly, the GF(1024)=GF(32²) extension `ζ²=ζ+1`, β, j_start=77, n≤93, `MD_REGULAR_CONST` as a wire constant). **FIREWALL: zero imports from `bch_decode.rs`.** Agreement set: since ≤4-error samples are already pinned by the Gap-1 sweep (both decoders must return the injected original), the reference decoder's MARGINAL value is on **beyond-t** words — include seeded e≥5 samples and assert md-codec ↔ reference agree on accept/reject AND on the miscorrected output (this pins the 29/4M re-verify class). RED-proof: an off-by-one in md-codec's syndrome window (`bch_decode.rs`) fails the reference-agreement test (now always-on).

## T2-c (#8) — mk `bch_correct_ok_implies_valid_codeword` proptest + third fuzz target
**Gap:** mk-codec `bch_correct_*` (`crates/mk-codec/src/string_layer/bch.rs`) — deleting the defensive re-verify (the F4-class guard) is caught by nothing; existing `bch_adversarial.rs` + `proptest_roundtrip.rs` don't pin the implication.
**Fix (test):** the defensive re-verify (mk `bch.rs:451`/`:504`) IS reachable but rare (R0 measured 29/4M ≈ **7.25e-6/trial** → ~2.9% RED at ≤4096 proptest cases). So combine an always-on proptest net with **mined deterministic KAT cells**:
- **proptest net (always-on invariant):** whenever `bch_correct_*` returns `Ok`, assert (i) result **verifies clean** (residue == target). Recast (iii) **UNCONDITIONAL** (as worded, an `Err` return vacuously satisfies an Ok-guarded implication → syndrome-window mutations survive): for ≤4 injected errors, `bch_correct_*` MUST return `Ok` AND == original. Pin (ii) **corrections ≤ 4** with an **exact-5-error KAT cell** (RED under relaxing `deg > 4` at mk `bch_decode.rs:566`).
- **mined re-verify KAT (deterministic RED-proof for implication (i)):** mine per-repo counterexamples against MK's own constants (`GEN_REGULAR`/`MK_REGULAR_CONST`, same ~2-min mining program the R0 used for md — which found e.g. inject `{(59,21),(92,7),(47,8),(44,26),(50,8),(11,8)}` → an L=3 correction that does NOT re-verify) — pin several as fixed cells. RED-proof: **deleting the post-correction re-verify** in `bch_correct_*` (`bch.rs:451`/`:504`) makes these cells emit an unverifiable `Ok` → implication (i) fails DETERMINISTICALLY (not the ~2.9%-flaky random path).
- **third fuzz target** (`fuzz/fuzz_targets/`, alongside `mk1_decode`/`mk1_decode_single`) — `mk1_bch_correct_never_invalid`: arbitrary bytes → `bch_correct_*`; on `Ok`, result MUST verify clean. Wire into `fuzz-smoke.yml` per the mk matrix + build-gate convention.
