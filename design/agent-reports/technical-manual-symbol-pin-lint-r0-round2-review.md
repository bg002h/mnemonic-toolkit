**C2 — §1's "top-level item" convention has no nesting rule; the real refs are overwhelmingly nested items, yielding either vacuous anchors or collapsing/non-discriminating ones, and making G2 a near-useless backstop.** §1 defines `<symbol>` = "the top-level item." But nearly every real ref points *inside* an item: methods in `impl` blocks, fns in `mod tests`, names in `pub use` blocks. The literal "top-level item" rule and the resolver's actual behaviour both break:

- **Methods → collapse + vacuous G2.** The audit queue + resolver are full of generic method guesses: `read`, `write`, `emit`, `new`, `fmt`, `as_bytes`, `decode`, `split`, `run`, `data` (~140 such resolutions across the manual, `cycle-b-resolver-output.tsv`). Distinct refs collapse to one anchor — e.g. `tlv.rs:134/158/184` all → `write` (`cycle-b-audit-queue.tsv:131-133`); `wallet_export/jade.rs` at four distinct ranges → `emit` ×4 (TSV 249-251, 258-260); `verify_bundle.rs:329` → `run` (TSV 225). `grep -wqF write` / `run` / `emit` passes against any file with such a method, so G2 gives ~zero existence assurance for the most common ref class.
- **Test modules.** Literal "top-level item" of a `mod tests` ref is `mod tests` → `file.rs::tests`, which is both useless documentation and a vacuous G2 token (every `.rs` file has a `mod tests`). Confirmed instances feeding the migration: `synthesize.rs:1732` (mod tests @ `synthesize.rs:828-829`), `chunk.rs:168` (mod tests @ `chunk.rs:88-89`), `json_envelope.rs:595/761` (mod tests @ `:473-474`). The resolver instead returns the *test fn* name — directly contradicting §1.
- **Re-exports.** `item_name` doesn't match `pub use`, so it scans up to the prior `pub mod`/item (the `lib.rs:43-50 → test_vectors` case in C1; `mod.rs:27-31` re-export blocks, TSV 150/154).

The SPEC must specify, per nesting context: method → `impl-target::method` or an explicit "method names are not unique; pin the impl-target" rule; test-module ref → either drop to bare `file.rs` or pin the test fn explicitly and *acknowledge G2 cannot discriminate*; re-export block → bare `file.rs` (the names listed are the anchor, not an enclosing item). Without this, both the audit and the auto-pin produce collapsing/vacuous anchors and G2 cannot back them.

**C3 — Cross-repo basename collision: chapters defaulting to `toolkit` cite codec basenames, and a colliding basename (`derive.rs`) resolves to the WRONG repo with G2 false-passing.** `default_repo` (`cycle-b-lineref-resolver.py:62-70`) routes chapters 30-39/40-49/54/60 to `toolkit`. But `derive.rs` exists in **both** `crates/mnemonic-toolkit/src/derive.rs` and `descriptor-mnemonic/crates/md-codec/src/derive.rs`, so the default-repo match wins and never falls back:
