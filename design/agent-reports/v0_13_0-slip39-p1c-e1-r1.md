## Verified clean against grep'd source

### Lagrange `interpolate_at` / `interpolate_secret_at` (`src/slip39/lagrange.rs`)

- Formula `Σ_i y_i · Π_{j≠i} (x XOR x_j) / (x_i XOR x_j)` (lines 52-53) matches SLIP-0039 §"Polynomial Interpolation" verbatim (subtraction in GF(2^8) collapses to XOR). Verified against python `shamir.py::_interpolate` @ `17fcce14`.
- Edge case `x == x_i`: toolkit lacks the python ref's short-circuit, but the algorithm is still correct — for the matching `i` the inner loop's `num/den` reduces to `Π (xi XOR x_j) / Π (xi XOR x_j) = 1` (i.e. yi survives); for non-matching `i'`, the factor `add(x, x_j_match) = 0` zeros out `num`. Result is mathematically equal to short-circuiting. SLIP-39's actual call sites are `x ∈ {0..15, 254, 255}` and `x_i ∈ {0..15}`, so collisions only occur for member-index interpolation — and the share-combine layer rejects duplicate member indices via `DuplicateMemberIndex` before reaching this layer. Safe.
- `interpolate_secret_at` panics on length mismatch (lines 73-82); R0 I1 notes the combine entry-point must catch this via `ShareValueLengthMismatch` before reaching lagrange. The variant is present and the driver (P1c-E.2 scope) is on the hook to use it.

### Feistel `extendable` routing (`src/slip39/feistel.rs`)

- 5th parameter `extendable: bool` added to `encrypt` and `decrypt`, threaded to `build_salt_prefix`. The `extendable=true` branch returns `Vec::new()`; `extendable=false` returns `b"shamir" || identifier_be(2)`. Byte-for-byte match against python `cipher.py::_get_salt(identifier, extendable)` @ `17fcce14`.
- All 19 P1b feistel tests had `, false` mechanically appended without flipping any assertion direction (R0 Q3 zero-tolerance constraint satisfied). 4 new ext=true tests added (round-trip 16B + 32B w/non-empty passphrase + cross-axis non-interop + ems-bytes-change-with-ext-flip). Cross-axis non-interop is structurally sound: salt prefixes differ across all 4 rounds → garbage out with overwhelming probability (~2^-128 for 16B master).

### `Slip39Error` Display + new variants (`src/slip39/error.rs`)

- All 5 new variants present (`EmptyShares`, `InvalidShareValueLength`, `ShareValueLengthMismatch`, `ExtendableMismatch`, `GroupThresholdExceedsCount`) with Display impls carrying the SPEC §2.5 stem placeholders. `InvalidShareValueLength { share_idx, got }` covers row 20 (I + L); `GroupThresholdExceedsCount { share_idx, threshold, count }` covers row 23 (J + T + N). - Carried `share_idx: 0` follows the single-share parser convention from P1c-D.
