## Seed-version dispatch (Electrum's `mnemonic.py::is_new_seed`)

```python
def is_new_seed(x: str, prefix=version.SEED_PREFIX) -> bool:
    x = normalize_text(x)
    s = hmac_oneshot(b"Seed version", x.encode('utf8'), hashlib.sha512).hex()
    return s.startswith(prefix)
```

Constants in `electrum/version.py`:

- `SEED_PREFIX = '01'` (standard)
- `SEED_PREFIX_SW = '100'` (segwit)
- `SEED_PREFIX_2FA = '101'` (2FA standard) — refused by md1 toolkit
- `SEED_PREFIX_2FA_SW = '102'` (2FA segwit) — refused by md1 toolkit

`normalize_text` is NFKD + lowercase + accent-strip + whitespace-collapse.

Reproducer (Python):

```python
import hmac, hashlib, unicodedata
def normalize(s):
    s = unicodedata.normalize('NFKD', s)
    s = s.lower()
    s = ''.join(c for c in s if not unicodedata.combining(c))
    s = ' '.join(s.split())
    return s
def hmac_prefix(phrase):
    return hmac.new(b'Seed version', normalize(phrase).encode('utf8'), hashlib.sha512).hexdigest()
```

## Encode-direction note (per SPEC §14)

Electrum's encode algorithm increments a starting integer until `validate_seed_version(generate_phrase(i)) == requested_version`. This means a user-supplied entropy hex value may NOT cleanly map to the requested `--electrum-version` on the first try — the toolkit must increment the integer until the HMAC prefix matches.

**Locked behavior (Phase 3 impl):** the encoder loops, incrementing the integer, until the HMAC prefix matches. The output phrase corresponds to the smallest `entropy + k` (k >= 0) whose phrase has the matching prefix. This is the canonical Electrum behavior. For the round-trip property test, this means `entropy_to_phrase(phrase_to_entropy(P, V), V) == P` holds for any phrase `P` whose decoded integer is itself the smallest matching (which is guaranteed for any phrase generated by Electrum or by this toolkit's encoder).

**Round-trip property:** `phrase_to_entropy → entropy_to_phrase` round-trips for any well-formed Electrum phrase. `entropy_to_phrase → phrase_to_entropy` round-trips when version is preserved.
