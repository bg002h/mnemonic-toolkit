## Verified clean against grep'd source + python ref

### `split_secret` (mod.rs) vs. python `_split_secret`

Verbatim cross-check against [`shamir.py::_split_secret` @ 17fcce14](https://github.com/trezor/python-shamir-mnemonic/blob/17fcce14/shamir_mnemonic/shamir.py):

- T==1 early-return: `(0..share_count).map(|i| (i, secret.to_vec())).collect()` matches python `[RawShare(i, shared_secret) for i in range(share_count)]`. No digest, no RNG draws.
- T>=2 random-share loop bound `for i in 0..(threshold - 2)` matches python `range(random_share_count)` where `random_share_count = threshold - 2`. For T==2, iterates 0 times (the R0 C1 foot-gun).
- `random_part` length = `n - DIGEST_LENGTH` (= `n - 4`), matches python `RANDOM_BYTES(len(shared_secret) - DIGEST_LENGTH_BYTES)`.
- `digest_payload = digest (4 bytes) || R (n-4 bytes)` ŌĆö `extend_from_slice(&digest_full[..DIGEST_LENGTH])` then `extend_from_slice(&r)`. Matches python `RawShare(DIGEST_INDEX, digest + random_part)`. R0 C1 concat-order foot-gun avoided.
- Base shares `[random_0..T-2, (254, digest_payload), (255, secret)]`. Matches python order.
- Emit loop: for `i < T-2` reuse the already-generated random share (avoids re-interpolation), for `i >= T-2` interpolate. Matches python's `for i in range(random_share_count, share_count): shares.append(RawShare(i, _interpolate(base_shares, i)))` ŌĆö the python ref starts emitting at index `random_share_count` because the random shares are already in `shares`; the rust code emits all `0..share_count` but skips re-interpolation for the random ones, producing the identical result.

### `recover_secret` (mod.rs) vs. python `_recover_secret`

- T==1 ŌåÆ `shares[0].1.clone()`; no digest. Matches python `next(iter(shares)).data`.
- T>=2 interpolates at 255 (secret) then 254 (digest_payload); splits digest_payload[..4] || digest_payload[4..]; HMAC verifies; refuses with `DigestVerificationFailed`. Verbatim against python.

### `slip39_split` (mod.rs) vs. python `generate_mnemonics` + `split_ems`

- Validation order: secret length ŌåÆ iteration_exponent ŌåÆ group_count + group_threshold ŌåÆ per-group spec ŌåÆ 1-of-N>1 policy refusal. The 1-of-N>1 refusal matches python `split_ems`'s `if any(mt == 1 and mc > 1 ...)`.
- Identifier mask `& 0x7FFF` produces a 15-bit identifier matching python `_random_identifier`.
- Feistel encrypt ŌåÆ group-level split ŌåÆ per-group member-level split ŌåÆ Share::from_parts. Field order matches `Share::from_parts` signature in share.rs.

### `slip39_combine` (mod.rs) ŌĆö 6-invariant order + group flow

- Step 1 empty refusal ŌåÆ `EmptyShares`.
- Step 3 per-share value-length: input-position `share_idx` (the I2 remap), refuses with `InvalidShareValueLength`.
- Step 4 6-invariant cross-share consistency checked in order: identifier ŌåÆ extendable ŌåÆ iter_exp ŌåÆ group_threshold ŌåÆ group_count ŌåÆ value_len. Matches the negative-vector mapping table (#6/#25 identifier, #7/#26 iter_exp, #8/#27 gt, #9/#28 gc). For vector #6, both shares start with `adequate`; word 2 differs (`smoking` vs `stay`); since `id_exp` spans words 1-2 and word 2 carries the low 5 bits of identifier + extendable + iter_exp, divergence at word 2 hits identifier as the FIRST check ŌĆö matches the test expectation.
- Step 5 BTreeMap group-by; member_threshold uniformity (first member as reference); distinct indices via sorted+windowed scan; strict-equal count; `recover_secret(mt, &pts)` per group.
- Step 6 strict-equal group-level threshold; `recover_secret(gt, &group_shares)`; feistel decrypt. Group-level sentinel `GROUP_LEVEL_SENTINEL = 255` is documented and outside the 4-bit group-index range.
