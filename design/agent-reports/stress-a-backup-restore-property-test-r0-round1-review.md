# R0 Review — stress-A backup-restore property test — ROUND 1

**Source SHA:** `3f4c66f`. **Verdict: 🔴 RED — 2 Critical / 3 Important / 5 Minor.**

The worst outcome for a property test is a GREEN test that proves nothing — this design had two such holes.

## Critical
**C1 — Pipeline seam broken.** `build-descriptor` emits a CONCRETE-keyed descriptor (gate step-2 `Descriptor::from_str` can't parse `@N`), but `bundle --descriptor <concrete> --slot @i.phrase=…` MISROUTES: `classify_descriptor_form` (`wallet_import/pipeline.rs:132-147`) sends a concrete descriptor to the **watch-only** path (`bundle.rs:1652`, slots ignored), the `@N` path hard-errors without placeholders (`bundle.rs:1183-1192`), mixing forms is refused. So the harness errors/misroutes ~100%. **Fix (recommended (a)):** concrete watch-only pipeline — generate the spec over an **origin-annotated** key pool (`[NNNNNNNN/48h/0h/0h/2h]xpub…`; `pipeline.rs:141-145` rejects origin-less concrete keys — gate-accept ≠ bundle-accept) → `build-descriptor --spec -` → `bundle --descriptor <desc>` NO slots → `restore --md1`. Original `desc` becomes the address-oracle source. **Failure policy: once the gate ACCEPTS, any bundle/restore failure is a PROPERTY FAILURE, never a `prop_assume!` skip** (the `<5%` guard only sees step-1 rejections).
**C2 — Oracle-3 near-vacuous.** `restore` computes `first_addresses` BY deriving from the reconstructed `desc'` (`restore.rs:1111→:1381`, `derive_receive_addresses`). Oracle-3 re-derives from the SAME `desc'` with the SAME patched miniscript → re-runs restore's own computation → catches nothing. The C1 collapse (wrong `wsh(multi)` + self-consistent wrong address) PASSES. **Fix:** derive the differential address from the ORIGINAL `desc` (source of truth) + assert == restore's reported `first_addresses[0]`; also derive `desc'` and assert it matches `desc`'s. Cover **chain-1 (change) + index ≥1**, not just receive/0. Depth-0 caveat doesn't break it (addresses depend only on chain_code+pubkey+suffix).

## Important
**I1 — Generator rejects ≫5%; typed-template should be PRIMARY.** Sanity≠typing: sigless branches (a timelock/hashlock alternative w/o a key on every path — `gate.rs:372-374`, the gate's own fixture is `or_d(pk,after(100))`); height+time timelock mixing (`gate.rs:382-384`); the d/u/n malleability lattice (`or_d` needs Bdu left, thresh subs Bdu/Wdu, `Malleable` `:375`); repeated keys (`:379-381` + bundle distinctness `:1665`). **Use typed-template PRIMARY** (sane schemas guaranteeing key-on-every-path + one timelock class per tree) + **fresh-key allocator** (no replacement) + `max_global_rejects = cases/20` + a **fragment-coverage accumulator** (≥N accepted cases each containing older/after/a-hash/multi/sortedmulti/thresh/or_*/andor — else coverage silently collapses, the canonicity-drift lesson).
**I2 — Oracle-1 substring holes.** `"multi(2,"` ⊂ `"sortedmulti(2,"` → multi→sortedmulti swap false-passes; positional spoofs (`older(1000)` elsewhere) pass. **Make Oracle-1 STRUCTURAL:** parse `desc`+`desc'` with miniscript, compare ASTs modulo key identity (erase keys to fingerprint slots; keep every k/timelock-value/digest/fragment-kind/arity/order).
**I3 — Permanent oracle negative cells.** The one-time "revert a v0.54.0 line → property fails+shrinks" bring-up proof is right (persist the transcript) but proves the harness ONCE. Add `#[test]` cells feeding each oracle a known-bad pair (older dropped; multi→sortedmulti; sha256→hash256; swapped or-branches; change-half multipath corruption) → assert the oracle REJECTS.
